# 前端国际化完善与后端日志聚合规范

## 背景与目标

本项目需要完善前端国际化（i18n）功能并增强后端日志聚合能力。前端目前仅支持英语和中文，需要扩展多语言支持并优化翻译加载机制；后端日志系统需要完善结构化格式、实现日志分级管理和敏感信息过滤。

## 变更内容

### 任务9: 前端国际化完善

#### 9.1 审计现有i18n覆盖范围

- **审计现有翻译模块**: common, nav, auth, dashboard, analytics, notifications, search, export
- **识别缺失翻译**: 用户设置、个人资料、表单验证等模块
- **评估翻译完整性**: 确保所有用户可见文本都有对应翻译

#### 9.2 添加更多语言支持

- **新增语言**: 日语(ja)、韩语(ko)、西班牙语(es)、法语(fr)、德语(de)
- **更新语言切换器**: 在 `LanguageSwitcher.jsx` 中添加新语言选项
- **持久化存储**: 使用 localStorage 保存用户语言偏好

#### 9.3 实现动态语言切换

- **即时切换**: 用户选择语言后立即生效，无需刷新页面
- **状态同步**: React 组件与 i18n 状态保持同步
- **回退机制**: 缺失翻译回退到英语

#### 9.4 添加日期时间本地化

- **date-fns 集成**: 在前端使用 date-fns 进行日期格式化
- **本地化格式**: 根据用户语言选择合适的日期格式
- **时区处理**: 支持不同时区的日期显示

#### 9.5 优化翻译文件加载

- **懒加载**: 使用 i18next 的懒加载功能按需加载翻译文件
- **缓存策略**: 浏览器缓存已加载的翻译资源
- **预加载**: 首屏加载默认语言翻译，后续语言按需加载

### 任务10: 后端日志聚合

#### 10.1 配置结构化日志格式

- **JSON 格式**: 所有日志输出为 JSON 格式便于解析
- **标准化字段**: timestamp, level, requestId, message, metadata
- **可读性输出**: 开发环境使用美化输出，生产环境使用压缩 JSON

#### 10.2 实现日志分级管理

- **日志级别**: error, warn, info, debug
- **环境控制**: 根据 NODE_ENV 控制日志输出级别
- **分级函数**: logError, logWarning, logInfo, logDebug

#### 10.3 添加请求追踪ID

- **请求ID生成**: 每个请求自动生成唯一追踪ID
- **上下文传播**: 将 requestId 传递到所有相关日志
- **响应头**: 在 HTTP 响应头中返回 X-Request-ID

#### 10.4 配置日志输出格式化

- **开发环境**: 彩色输出，易读的格式
- **生产环境**: 压缩 JSON，单行输出
- **日志文件**: 支持输出到文件（可选）

#### 10.5 添加敏感信息过滤

- **敏感字段**: password, token, secret, apiKey, creditCard
- **过滤机制**: 在日志输出前自动替换敏感信息
- **可配置列表**: 支持添加自定义敏感字段

## 功能需求

### 需求: 多语言支持

系统应支持至少7种语言：英语、中文、日语、韩语、西班牙语、法语、德语。

#### 场景: 语言切换

- **当** 用户在语言切换器中选择新语言
- **则** 页面立即更新为新语言，localStorage 保存用户偏好

### 需求: 日期时间本地化

系统应根据用户语言显示格式化的日期时间。

#### 场景: 日期显示

- **当** 用户使用日语界面
- **则** 日期显示为 "2024年1月15日" 格式

### 需求: 结构化日志

后端应输出符合 JSON 格式的结构化日志。

#### 场景: 日志记录

- **当** 发生错误时
- **则** 输出包含 requestId, timestamp, level, message, stack 等字段的 JSON 日志

### 需求: 敏感信息过滤

日志中不应包含密码、令牌等敏感信息。

#### 场景: 敏感数据保护

- **当** 日志记录包含密码字段的请求
- **则** 密码值被替换为 `[REDACTED]`

## 技术方案

### 前端国际化

1. **i18next 配置**: 使用 react-i18next 和 i18next
2. **日期本地化**: 使用 date-fns 的 locale 功能
3. **状态管理**: i18n.changeLanguage 动态切换
4. **持久化**: localStorage 存储语言偏好

### 后端日志

1. **日志格式**: 自定义 JSON 格式化器
2. **日志级别**: 通过环境变量 LOG_LEVEL 控制
3. **敏感信息**: 使用正则表达式过滤敏感字段
4. **请求追踪**: 使用 uuid 或自定义格式生成 requestId
