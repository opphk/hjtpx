# 开发任务规划 - 行为验证系统 v2.0

## 项目概述
开发一个完整的行为验证系统，前后端均使用Go语言，目标是超越极验、易盾、五秒盾等现有产品。

## 技术规范
- **CSS使用**: 所有CSS必须从bootcdn.cn加载，禁止编写自定义CSS
- **CDN资源**:
  - Bootstrap 5: `https://cdn.bootcdn.net/ajax/libs/twitter-bootstrap/5.3.8/css/bootstrap.min.css`
  - Bootstrap 5 JS: `https://cdn.bootcdn.net/ajax/libs/twitter-bootstrap/5.3.8/js/bootstrap.bundle.min.js`
  - Font Awesome 6: `https://cdn.bootcdn.net/ajax/libs/font-awesome/6.5.1/css/all.min.css`
  - Chart.js: `https://cdn.bootcdn.net/ajax/libs/Chart.js/4.4.0/chart.umd.min.js`

## 开发优先级
1. 高优先级: 任务1-4 (核心验证码功能)
2. 中优先级: 任务5-7 (用户和应用管理)
3. 高优先级: 任务8-9 (安全和限流)
4. 中优先级: 任务10-11 (前端和管理端)
5. 低优先级: 任务12-15 (优化和文档)

## 里程碑
- **M1**: 基础验证码功能完善 (任务1-3)
- **M2**: 行为分析与安全防护 (任务4, 8-9)
- **M3**: 用户与管理功能 (任务5-7, 10-11)
- **M4**: 部署与测试 (任务12-15)
