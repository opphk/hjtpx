## 测试概览

v19.0 引入了三个新的测试类别：

| 测试类型 | 描述 | 优先级 |
|---------|------|--------|
| **模糊测试 (Fuzz Testing) | 自动生成随机输入发现边界情况和崩溃 | 高 |
| **混沌工程测试 (Chaos Engineering) | 在系统中注入故障测试弹性 | 高 |

### 模糊测试

模糊测试用于发现边缘情况和潜在的崩溃。

#### 核心功能

- 随机输入生成
- 边界情况测试
- 自动崩溃检测

#### 使用方法

```go
import "github.com/hjtpx/hjtpx/internal/testing/fuzz"

// 创建模糊测试器
fuzzer := fuzz.NewFuzzer(&fuzz.FuzzingConfig{
    Iterations: 10000,
    Seed:       42,
    Verbose:    true,
})

// 运行模糊测试
result := fuzzer.Run(t, "MyTest", func(input []byte) error {
    // 测试代码
    return nil
})
```

#### 运行模糊测试

```bash
make test-fuzz
```

### 混沌工程测试

混沌工程用于测试系统在面对故障时的表现。

#### 核心功能

- 延迟注入
- 故障注入
- 网络分区
- 资源耗尽

#### 使用方法

```go
import "github.com/hjtpx/hjtpx/internal/testing/chaos"

// 创建混沌引擎
engine := chaos.NewChaosEngine()

// 添加实验
engine.AddExperiment(&chaos.ChaosExperiment{
    Name:        "API Latency",
    Description: "Inject latency into API calls",
    Setup: func() error {
        return nil
    },
    Run: func(ctx context.Context) error {
        injector := chaos.NewLatencyInjector(100*time.Millisecond, 50*time.Millisecond)
        injector.Inject()
        return nil
    },
})

// 运行实验
result, err := engine.RunExperiment(ctx, "API Latency")
```

#### 运行混沌测试

```bash
make test-chaos
```

### 安全测试

安全测试基于 OWASP Top 10 2021。

#### 核心功能

- SQL注入检测
- XSS检测
- 命令注入检测
- 路径遍历检测
- XXE检测
- SSRF检测
- 密码强度检测
- 输入清理

#### 使用方法

```go
import "github.com/hjtpx/hjtpx/internal/testing/security"

// 创建安全扫描器
scanner := security.NewSecurityScanner()

// 扫描所有漏洞
vulns := scanner.ScanAll(userInput)

// 检查密码强度
strength := security.CheckPasswordStrength(password)

// 清理输入
sanitized := security.SanitizeInput(input)
```

#### 运行安全测试

```bash
make test-security
```

## 测试最佳实践

### 模糊测试

1. **从简单开始**：先用较少迭代开始，逐步增加复杂度
2. **使用固定种子**：在调试时使用固定种子以便重现问题
3. **记录崩溃输入**：保存导致崩溃的输入以便修复和回归测试

### 混沌工程

1. **在测试环境进行**：不要在生产环境进行混沌实验
2. **从小规模开始**：先从小规模实验开始
3. **有快速恢复**：确保有快速恢复机制

### 安全测试

1. **持续扫描**：将安全测试集成到 CI/CD 流程中
2. **假阳性处理**：定期审查和处理误报
3. **及时修复**：优先处理发现的漏洞

## 故障排查

### 测试失败

查看测试输出以定位问题：

```bash
# 查看详细日志
make test-fuzz 2>&1 | tee fuzz.log
```

### 模糊测试崩溃

保存崩溃输入并用固定种子重试：

```go
config := &fuzz.FuzzingConfig{
    Seed: 42, // 使用导致崩溃的种子
    Iterations: 1,
}
```
