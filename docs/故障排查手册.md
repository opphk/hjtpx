## 引言

本手册旨在帮助您快速识别和解决 HJTPX 验证系统运行过程中可能遇到的问题。通过系统性的排查流程，您可以高效地定位并修复故障。

### 排查原则

1. **先观察，后行动**：收集足够的信息，不要盲目修改配置
2. **从简单到复杂**：先检查常见问题，再深入复杂问题
3. **单一变量**：每次只修改一个参数，便于定位问题
4. **记录过程**：记录所有操作和结果，便于回溯和总结
5. **验证修复**：问题解决后，必须验证服务已恢复正常

## 排查方法

### 1. 状态检查

#### 服务状态

```bash
# Docker Compose 部署
docker-compose ps

# 检查容器健康状态
docker-compose ps --format json

# 检查系统服务
systemctl status hjtpx
```

#### 健康检查

```bash
# 应用健康检查
curl http://localhost:8080/health

# 检查 HTTP 状态码
curl -I http://localhost:8080/health
```

### 2. 日志收集

```bash
# 查看所有服务日志
docker-compose logs --tail=100

# 实时跟踪日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f postgres
docker-compose logs -f redis

# 搜索错误日志
docker-compose logs backend | grep -i error
docker-compose logs backend | grep -i warn

# 查看最近 N 行日志
docker-compose logs --tail=500 backend
```

### 3. 指标收集

```bash
# 检查容器资源使用
docker stats

# 查看 Prometheus 指标
curl http://localhost:9090/metrics

# 检查数据库连接
docker-compose exec postgres psql -U postgres -c "SELECT count(*) FROM pg_stat_activity;"

# 检查 Redis 信息
docker-compose exec redis redis-cli info
```

### 4. 系统诊断流程

```
问题发现
  │
  ├─→ 收集信息（日志、指标、状态）
  │
  ├─→ 确定范围（应用/数据库/Redis/网络）
  │
  ├─→ 排查原因（检查配置、依赖、资源）
  │
  ├─→ 尝试修复（单一变量、记录过程）
  │
  ├─→ 验证修复（测试功能、监控指标）
  │
  └─→ 总结记录（问题原因、解决方案）
```

## 常见问题与解决方案

### 服务启动问题

#### 问题 1：容器无法启动

**症状**：
- `docker-compose ps` 显示容器状态为 `Exited`
- 日志显示启动错误

**排查步骤**：

1. 查看详细日志：
   ```bash
   docker-compose logs backend
   ```

2. 检查端口占用：
   ```bash
   netstat -tlnp | grep 8080
   lsof -i :8080
   ```

3. 检查配置文件：
   ```bash
   docker-compose config
   ```

4. 检查依赖服务是否就绪：
   ```bash
   docker-compose logs postgres
   docker-compose logs redis
   ```

**常见原因与解决方案**：

| 原因 | 解决方案 |
|------|----------|
| 端口被占用 | 修改端口配置或停止占用端口的进程 |
| 数据库连接失败 | 检查数据库地址、端口、用户名、密码 |
| 配置文件错误 | 检查 `config.yaml` 格式和内容 |
| 内存不足 | 增加容器内存限制或清理系统内存 |
| 镜像拉取失败 | 检查网络连接，或重新拉取镜像 |

#### 问题 2：服务启动但无法访问

**症状**：
- 容器状态显示 `Up`
- 但无法通过 HTTP 访问

**排查步骤**：

1. 检查端口映射：
   ```bash
   docker-compose port backend 8080
   ```

2. 检查防火墙：
   ```bash
   iptables -L -n
   ufw status
   ```

3. 检查应用日志：
   ```bash
   docker-compose logs backend | grep -i "listening\|starting"
   ```

4. 从容器内部测试：
   ```bash
   docker-compose exec backend curl http://localhost:8080/health
   ```

### 连接问题

#### 问题 3：数据库连接失败

**症状**：
- 日志显示 `connection refused` 或 `timeout`
- 健康检查显示数据库状态为 `down`

**排查步骤**：

1. 检查数据库容器状态：
   ```bash
   docker-compose ps postgres
   ```

2. 测试网络连通性：
   ```bash
   docker-compose exec backend ping postgres
   docker-compose exec backend nc -zv postgres 5432
   ```

3. 检查数据库日志：
   ```bash
   docker-compose logs postgres
   ```

4. 手动连接测试：
   ```bash
   docker-compose exec postgres psql -U postgres -d hjtpx_db
   ```

**常见原因与解决方案**：

| 原因 | 解决方案 |
|------|----------|
| 数据库未启动 | 启动数据库容器：`docker-compose start postgres` |
| 网络问题 | 检查 Docker 网络，重启容器 |
| 认证失败 | 检查用户名、密码，重置数据库密码 |
| 连接数过多 | 增加数据库最大连接数，或优化应用连接池 |

#### 问题 4：Redis 连接失败

**症状**：
- 缓存功能失效
- 日志显示 Redis 连接错误

**排查步骤**：

1. 检查 Redis 容器状态：
   ```bash
   docker-compose ps redis
   ```

2. 测试 Redis 连接：
   ```bash
   docker-compose exec redis redis-cli ping
   ```

3. 检查 Redis 日志：
   ```bash
   docker-compose logs redis
   ```

4. 查看 Redis 信息：
   ```bash
   docker-compose exec redis redis-cli info
   ```

### 功能问题

#### 问题 5：验证码生成失败

**症状**：
- 前端显示验证码加载失败
- 接口返回 500 错误

**排查步骤**：

1. 检查应用日志：
   ```bash
   docker-compose logs backend | grep -i captcha
   ```

2. 测试验证码接口：
   ```bash
   curl -v http://localhost:8080/api/v1/captcha/image
   ```

3. 检查 Redis 中是否有验证码数据：
   ```bash
   docker-compose exec redis redis-cli keys "captcha:*"
   ```

4. 检查静态文件服务：
   ```bash
   curl -I http://localhost/static/
   ```

#### 问题 6：验证功能异常

**症状**：
- 验证码验证总是失败
- 验证接口超时或返回错误

**排查步骤**：

1. 查看验证日志：
   ```bash
   docker-compose logs backend | grep -i verify
   ```

2. 测试完整流程：
   ```bash
   # 1. 获取验证码
   CAPTCHA_ID=$(curl -s http://localhost:8080/api/v1/captcha/image | jq -r .captcha_id)
   
   # 2. 假设验证码是 1234（实际需要从响应中获取或用户输入）
   # 3. 提交验证
   curl -X POST http://localhost:8080/api/v1/captcha/verify \
     -H "Content-Type: application/json" \
     -d "{\"captcha_id\": \"$CAPTCHA_ID\", \"answer\": \"1234\"}"
   ```

3. 检查数据库中验证记录：
   ```bash
   docker-compose exec postgres psql -U postgres -d hjtpx_db \
     -c "SELECT * FROM verification_logs ORDER BY created_at DESC LIMIT 10;"
   ```

## 日志分析

### 日志位置

| 服务 | 日志路径 | Docker 查看方式 |
|------|----------|-----------------|
| 应用 | `/var/log/hjtpx/app.log` | `docker-compose logs backend` |
| Nginx | `/var/log/nginx/` | `docker-compose logs nginx` |
| PostgreSQL | Docker 容器日志 | `docker-compose logs postgres` |
| Redis | Docker 容器日志 | `docker-compose logs redis` |

### 日志级别

| 级别 | 含义 | 示例 |
|------|------|------|
| DEBUG | 调试信息 | 详细的请求处理过程 |
| INFO | 一般信息 | 服务启动、请求成功 |
| WARN | 警告信息 | 可能的问题但不影响功能 |
| ERROR | 错误信息 | 需要处理的问题 |
| FATAL | 致命错误 | 服务无法继续运行 |

### 常见错误日志分析

#### 1. 数据库相关错误

```
ERROR: connection to postgres:5432 refused
```
→ 检查数据库是否启动，网络连接是否正常

```
ERROR: password authentication failed
```
→ 检查数据库用户名和密码

```
ERROR: too many connections
```
→ 检查数据库连接数限制，优化连接池配置

#### 2. Redis 相关错误

```
ERROR: redis: connection refused
```
→ 检查 Redis 是否启动，地址和端口配置

```
ERROR: OOM command not allowed
```
→ Redis 内存不足，增加内存或配置淘汰策略

#### 3. 应用相关错误

```
ERROR: invalid captcha
```
→ 验证码过期或错误

```
ERROR: rate limit exceeded
```
→ 请求频率超限，检查速率限制配置

## 性能问题

### 响应缓慢

**排查步骤**：

1. 检查系统资源使用：
   ```bash
   top
   htop
   docker stats
   ```

2. 检查数据库慢查询：
   ```sql
   -- PostgreSQL 慢查询
   SELECT query, calls, mean_time, total_time
   FROM pg_stat_statements
   ORDER BY mean_time DESC
   LIMIT 10;
   ```

3. 检查应用日志中的慢请求：
   ```bash
   docker-compose logs backend | grep -i "slow\|latency"
   ```

4. 使用 curl 测试响应时间：
   ```bash
   curl -w "@curl-format.txt" -o /dev/null -s http://localhost:8080/health
   ```

### 吞吐量低

**排查步骤**：

1. 检查连接池配置：
   ```yaml
   postgres:
     max_open_conns: 100
     max_idle_conns: 20
   
   redis:
     pool_size: 100
   ```

2. 检查数据库连接数：
   ```sql
   SELECT count(*) FROM pg_stat_activity;
   ```

3. 检查 Redis 连接数：
   ```bash
   docker-compose exec redis redis-cli info clients
   ```

4. 检查 Nginx 连接数：
   ```bash
   curl http://localhost/nginx_status
   ```

## 数据库问题

### PostgreSQL 常见问题

#### 问题 7：数据库查询慢

**排查步骤**：

1. 启用并查看慢查询日志：
   ```sql
   -- 检查慢查询配置
   SHOW log_min_duration_statement;
   
   -- 设置慢查询阈值（毫秒）
   SET log_min_duration_statement = 100;
   ```

2. 分析查询执行计划：
   ```sql
   EXPLAIN ANALYZE SELECT * FROM verification_logs WHERE created_at > NOW() - INTERVAL '1 day';
   ```

3. 检查索引使用情况：
   ```sql
   SELECT
     schemaname,
     tablename,
     indexname,
     idx_scan,
     idx_tup_read,
     idx_tup_fetch
   FROM pg_stat_user_indexes
   ORDER BY idx_scan DESC;
   ```

4. 检查表大小：
   ```sql
   SELECT
     schemaname,
     tablename,
     pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
   FROM pg_tables
   WHERE schemaname = 'public'
   ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;
   ```

**优化方案**：
- 添加适当的索引
- 优化 SQL 查询
- 定期清理或归档历史数据
- 考虑读写分离

#### 问题 8：数据库空间不足

**排查步骤**：

1. 检查磁盘使用：
   ```bash
   df -h
   ```

2. 检查表和索引大小：
   ```sql
   SELECT
     schemaname || '.' || tablename AS table,
     pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS total_size,
     pg_size_pretty(pg_relation_size(schemaname||'.'||tablename)) AS table_size,
     pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename) - pg_relation_size(schemaname||'.'||tablename)) AS index_size
   FROM pg_tables
   WHERE schemaname = 'public'
   ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;
   ```

3. 检查并清理 WAL 文件：
   ```bash
   # 查看 WAL 文件大小
   ls -lh /var/lib/postgresql/data/pg_wal/ | wc -l
   ```

**解决方案**：
- 清理临时文件
- 归档历史数据
- 增加磁盘空间
- 调整 WAL 保留策略

## Redis 问题

### Redis 常见问题

#### 问题 9：Redis 内存使用过高

**排查步骤**：

1. 查看 Redis 内存使用：
   ```bash
   docker-compose exec redis redis-cli info memory
   ```

2. 查看大 Key：
   ```bash
   # 使用 redis-cli --bigkeys
   docker-compose exec redis redis-cli --bigkeys
   ```

3. 查看 Key 空间分布：
   ```bash
   docker-compose exec redis redis-cli info keyspace
   ```

**解决方案**：
- 配置内存淘汰策略
- 清理过期或无用的 Key
- 增加 Redis 内存限制
- 考虑 Redis 集群

#### 问题 10：Redis 持久化问题

**排查步骤**：

1. 检查持久化配置：
   ```bash
   docker-compose exec redis redis-cli config get save
   docker-compose exec redis redis-cli config get appendonly
   ```

2. 查看持久化日志：
   ```bash
   docker-compose logs redis | grep -i "rdb\|aof"
   ```

3. 检查 RDB/AOF 文件：
   ```bash
   docker-compose exec redis ls -lh /data/
   ```

**解决方案**：
- 调整持久化策略
- 检查磁盘空间
- 修复 AOF 文件（如需要）：
  ```bash
  redis-check-aof --fix appendonly.aof
  ```

## 网络问题

### 网络排查工具

```bash
# 检查网络连通性
ping <host>
traceroute <host>

# 检查端口连通性
telnet <host> <port>
nc -zv <host> <port>

# 检查 DNS 解析
nslookup <host>
dig <host>

# 查看网络连接
netstat -tlnp
ss -tlnp

# 查看路由表
ip route
route -n
```

### 常见网络问题

#### 问题 11：容器间网络不通

**排查步骤**：

1. 检查 Docker 网络：
   ```bash
   docker network ls
   docker network inspect hjtpx_default
   ```

2. 检查容器 IP：
   ```bash
   docker-compose exec backend hostname -i
   docker-compose exec postgres hostname -i
   ```

3. 从一个容器 ping 另一个容器：
   ```bash
   docker-compose exec backend ping postgres
   ```

#### 问题 12：外部无法访问服务

**排查步骤**：

1. 检查端口映射：
   ```bash
   docker-compose port backend 8080
   ```

2. 检查防火墙：
   ```bash
   sudo iptables -L -n -v
   sudo ufw status
   ```

3. 检查云服务商安全组（如适用）。
