# 第三方集成配置说明

## 1. OAuth2 集成

### 1.1 GitHub OAuth2 配置

```yaml
oauth2:
  github:
    client_id: "your-github-client-id"
    client_secret: "your-github-client-secret"
    redirect_uri: "https://your-domain.com/api/v1/oauth2/callback"
    scopes:
      - "read:user"
      - "user:email"
```

### 1.2 Google OAuth2 配置

```yaml
oauth2:
  google:
    client_id: "your-google-client-id.apps.googleusercontent.com"
    client_secret: "your-google-client-secret"
    redirect_uri: "https://your-domain.com/api/v1/oauth2/callback"
    scopes:
      - "openid"
      - "email"
      - "profile"
```

### 1.3 OAuth2 API 接口

#### 获取支持的提供商
```
GET /api/v1/oauth2/providers
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "providers": ["github", "google"],
    "enabled": {
      "github": true,
      "google": true
    }
  }
}
```

#### 发起OAuth2登录
```
GET /api/v1/oauth2/login/:provider
```

参数：
- `provider`: `github` 或 `google`

响应示例：
```json
{
  "code": 0,
  "data": {
    "auth_url": "https://github.com/login/oauth/authorize?...",
    "state": "random-state-string",
    "provider": "github",
    "expires_in": 600
  }
}
```

#### 处理OAuth2回调
```
GET /api/v1/oauth2/callback?code=xxx&state=xxx
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "provider": "github",
    "provider_id": "123456",
    "username": "user",
    "email": "user@example.com",
    "name": "User Name",
    "avatar_url": "https://avatars.githubusercontent.com/u/123456",
    "access_token": "gho_xxx",
    "token_type": "Bearer",
    "expires_in": 3600
  }
}
```

#### 刷新Token
```
POST /api/v1/oauth2/refresh
```

请求体：
```json
{
  "provider": "github",
  "refresh_token": "xxx"
}
```

#### 撤销Token
```
POST /api/v1/oauth2/revoke
```

请求体：
```json
{
  "provider": "github",
  "token": "xxx"
}
```

## 2. SSO 单点登录

### 2.1 SAML 配置

```yaml
sso:
  saml:
    myidp:
      entity_id: "https://idp.example.com"
      sso_url: "https://idp.example.com/sso"
      slo_url: "https://idp.example.com/slo"
      certificate: "base64-encoded-certificate"
      sp_entity_id: "https://your-domain.com"
      acs_url: "https://your-domain.com/api/v1/sso/saml/callback"
      name_id_format: "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress"
```

### 2.2 CAS 配置

```yaml
sso:
  cas:
    mycas:
      server_url: "https://cas.example.com"
      service_url: "https://your-domain.com"
      version: "2.0"
      allow_proxy: false
```

### 2.3 SSO API 接口

#### 获取SSO提供商
```
GET /api/v1/sso/providers
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "saml_providers": ["myidp"],
    "cas_providers": ["mycas"]
  }
}
```

#### SAML登录
```
GET /api/v1/sso/saml/login/:provider
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "provider": "myidp",
    "request_id": "_1234567890",
    "auth_url": "https://idp.example.com/sso",
    "encoded_request": "base64-encoded-saml-request"
  }
}
```

#### SAML回调
```
POST /api/v1/sso/saml/callback
```

表单数据：
- `SAMLResponse`: Base64编码的SAML响应
- `provider`: 提供商名称

#### CAS登录
```
GET /api/v1/sso/cas/login/:provider
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "provider": "mycas",
    "login_url": "https://cas.example.com/login?service=https://your-domain.com"
  }
}
```

#### CAS回调
```
GET /api/v1/sso/cas/callback?provider=mycas&ticket=ST-xxx
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "provider": "cas",
    "provider_id": "ST-xxx",
    "username": "user",
    "email": "user@example.com",
    "attributes": {...},
    "session_id": "cas_user_1234567890"
  }
}
```

#### SSO状态
```
GET /api/v1/sso/status
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "authenticated": true,
    "provider": "cas",
    "username": "user",
    "email": "user@example.com",
    "session_id": "cas_user_1234567890"
  }
}
```

#### 登出
```
POST /api/v1/sso/logout
GET /api/v1/sso/logout
```

## 3. Webhook 支持

### 3.1 Webhook 配置

```yaml
webhooks:
  signature_key: "your-webhook-secret-key"
```

### 3.2 Webhook API 接口

#### 注册Webhook端点
```
POST /api/v1/webhooks/endpoints
```

请求体：
```json
{
  "name": "My Webhook",
  "url": "https://my-app.com/webhook",
  "secret": "webhook-secret",
  "event_types": [
    "verification.success",
    "verification.failed",
    "risk.detected"
  ],
  "headers": {
    "X-Custom-Header": "value"
  },
  "timeout": 10,
  "retry_policy": {
    "max_retries": 3,
    "initial_delay": 1,
    "max_delay": 60,
    "backoff_factor": 2.0,
    "retryable_codes": [408, 429, 500, 502, 503, 504]
  },
  "active": true
}
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "id": "wh_xxx",
    "name": "My Webhook",
    "url": "https://my-app.com/webhook",
    "active": true,
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

#### 列出端点
```
GET /api/v1/webhooks/endpoints
```

#### 获取端点
```
GET /api/v1/webhooks/endpoints/:id
```

#### 更新端点
```
PUT /api/v1/webhooks/endpoints/:id
```

#### 删除端点
```
DELETE /api/v1/webhooks/endpoints/:id
```

#### 测试端点
```
POST /api/v1/webhooks/endpoints/:id/test
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "delivery_id": "wh_del_xxx",
    "status": "delivered",
    "response_code": 200,
    "response_body": "OK"
  }
}
```

#### 触发事件
```
POST /api/v1/webhooks/trigger
```

请求体：
```json
{
  "event": "verification.success",
  "data": {
    "user_id": "123",
    "captcha_type": "slider",
    "duration_ms": 1500
  },
  "ip_address": "1.2.3.4",
  "user_agent": "Mozilla/5.0...",
  "request_id": "req_xxx"
}
```

#### 获取投递记录
```
GET /api/v1/webhooks/deliveries?endpoint_id=xxx&limit=100
```

#### 重试投递
```
POST /api/v1/webhooks/deliveries/:id/retry
```

#### 获取统计
```
GET /api/v1/webhooks/stats
```

响应示例：
```json
{
  "code": 0,
  "data": {
    "total_endpoints": 5,
    "active_endpoints": 4,
    "total_deliveries": 1000,
    "delivered": 950,
    "failed": 50,
    "pending": 0
  }
}
```

### 3.3 Webhook 签名验证

Webhook 请求包含以下头部：
- `X-Webhook-ID`: 投递ID
- `X-Webhook-Event`: 事件类型
- `X-Webhook-Timestamp`: 时间戳
- `X-Webhook-Signature`: HMAC-SHA256签名

签名计算方式：
```
signature = "sha256=" + hex(hmac_sha256(secret, timestamp + "." + payload))
```

## 4. Slack 通知集成

### 4.1 Slack 配置

```yaml
alerts:
  channels:
    slack:
      webhook_url: "https://hooks.slack.com/services/xxx/yyy/zzz"
      channel: "#alerts"
      username: "HJTPX Alerts"
      icon_emoji: ":warning:"
      min_severity: "warning"
      allowed_severities: []
      blocked_severities: ["debug"]
      retry_enabled: true
      max_retries: 3
```

### 4.2 Slack API 使用示例

```go
// 创建Slack渠道
config := map[string]interface{}{
    "webhook_url": "https://hooks.slack.com/services/xxx",
    "channel": "#alerts",
    "username": "HJTPX",
    "icon_emoji": ":robot_face:",
    "min_severity": "info",
}

channel, err := service.NewSlackEnhancedChannel(config)

// 发送告警
msg := service.AlertMessage{
    Title:     "验证失败",
    Message:   "检测到异常验证行为",
    Severity:  "warning",
    EventID:   "evt_123",
    Timestamp: time.Now(),
    Context: map[string]interface{}{
        "user_id":    "123",
        "ip_address": "1.2.3.4",
        "risk_score": 0.85,
    },
}

err = channel.Send(msg)

// 发送富消息
err = channel.SendRichMessage(
    "新用户注册",
    "检测到新用户注册",
    "info",
    map[string]string{
        "用户":   "user@example.com",
        "IP":    "1.2.3.4",
        "时间":  "2024-01-01 12:00:00",
    },
    "",
)
```

## 5. 企业微信通知集成

### 5.1 企业微信配置

#### 通过Webhook方式
```yaml
alerts:
  channels:
    wechat_work:
      webhook_url: "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx"
      min_severity: "error"
```

#### 通过API方式
```yaml
alerts:
  channels:
    wechat_work_api:
      corp_id: "wwxxxxx"
      corp_secret: "xxxxx"
      agent_id: "1000001"
      to_users:
        - "user1"
        - "user2"
      to_parties:
        - "party1"
```

### 5.2 企业微信 API 使用示例

```go
// 创建企业微信渠道
config := map[string]interface{}{
    "webhook_url": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx",
    "min_severity": "info",
}

channel, err := service.NewWeChatWorkEnhancedChannel(config)

// 发送告警
msg := service.AlertMessage{
    Title:     "系统告警",
    Message:   "CPU使用率超过90%",
    Severity:  "critical",
    EventID:   "evt_456",
    Timestamp: time.Now(),
    Context: map[string]interface{}{
        "server":    "prod-01",
        "cpu_usage": "95%",
        "memory":    "82%",
    },
}

err = channel.Send(msg)

// 发送文本消息
err = channel.SendTextMessage(
    "这是一条测试消息",
    []string{"user1", "user2"},
    []string{"party1"},
)

// 发送Markdown消息
markdown := `### 告警通知

> **服务器**: prod-01
> **CPU**: 95%
> **内存**: 82%
`

err = channel.SendMarkdownMessage(markdown)

// 发送图文消息
articles := []service.WeChatArticle{
    {
        Title:       "告警详情",
        Description: "查看告警详细信息",
        URL:         "https://your-domain.com/alerts/123",
        PicURL:      "https://your-domain.com/images/alert.png",
    },
}

err = channel.SendNewsMessage(articles)
```

## 6. 错误级别过滤

所有通知渠道都支持错误级别过滤：

```yaml
min_severity: "warning"        # 只发送 warning 及以上级别
allowed_severities:            # 只发送指定级别
  - "error"
  - "critical"
blocked_severities:            # 排除指定级别
  - "debug"
  - "info"
```

### 严重级别排序（从低到高）

1. debug
2. info
3. notice
4. warning/warn
5. error
6. critical
7. alert
8. emergency/emerg

## 7. 最佳实践

### 7.1 安全建议

1. **OAuth2 Client Secret**
   - 不要将client_secret硬编码在代码中
   - 使用环境变量或密钥管理服务
   - 定期轮换密钥

2. **Webhook Secret**
   - 为每个Webhook端点使用独立的secret
   - 验证Webhook签名以确保请求真实性
   - 使用HTTPS传输

3. **SSO证书**
   - 定期更新SAML签名证书
   - 使用有效的证书链

### 7.2 性能优化

1. **Webhook**
   - 设置合理的超时时间
   - 启用重试机制
   - 使用异步投递

2. **通知渠道**
   - 配置适当的重试次数
   - 使用连接池
   - 实现批量发送

### 7.3 监控和日志

1. 监控所有集成端点的可用性
2. 记录所有失败的投递
3. 设置告警阈值
4. 定期检查和清理旧的投递记录
