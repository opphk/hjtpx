## 环境要求

### 硬件要求

| 部署规模 | CPU | 内存 | 磁盘 | 网络 |
|---------|-----|------|------|------|
| 开发/测试 | 2核 | 4GB | 20GB | 100Mbps |
| 小型生产（<100 QPS） | 4核 | 8GB | 50GB | 1Gbps |
| 中型生产（100-1000 QPS） | 8核 | 16GB | 100GB | 1Gbps |
| 大型生产（>1000 QPS） | 16核+ | 32GB+ | 500GB+ | 10Gbps |

### 软件要求

| 组件 | 最低版本 | 推荐版本 | 说明 |
|------|---------|---------|------|
| Go | 1.21+ | 1.25+ | 仅手动部署需要 |
| PostgreSQL | 12+ | 16+ | 关系型数据库 |
| Redis | 6+ | 7+ | 缓存和会话存储 |
| Nginx | 1.18+ | 1.25+ | 反向代理（可选） |
| Docker | 20.10+ | 24.0+ | 容器化部署 |
| Kubernetes | 1.24+ | 1.28+ | 容器编排（可选） |

### 操作系统支持

- Linux: Ubuntu 20.04+, Debian 11+, CentOS 8+, RHEL 8+
- macOS: 11+ (仅限开发环境)
- Windows: 10+ (仅限开发环境，推荐使用 WSL2)

## Docker 部署

### 基础部署

#### 方式一：使用官方镜像

```bash
# 拉取镜像
docker pull ghcr.io/opphk/hjtpx:latest

# 运行容器
docker run -d \
  --name hjtpx-app \
  -p 8080:8080 \
  -e POSTGRES_HOST=your-postgres-host \
  -e POSTGRES_PORT=5432 \
  -e POSTGRES_DB=hjtpx_db \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=your-password \
  -e REDIS_HOST=your-redis-host \
  -e REDIS_PORT=6379 \
  -e REDIS_PASSWORD=your-redis-password \
  -e JWT_SECRET=your-very-secure-jwt-secret-key \
  ghcr.io/opphk/hjtpx:latest
```

#### 方式二：自行构建镜像

```bash
# 克隆代码
git clone https://github.com/opphk/hjtpx.git
cd hjtpx

# 构建镜像
docker build -t hjtpx:latest .

# 运行容器
docker run -d \
  --name hjtpx-app \
  -p 8080:8080 \
  --network hjtpx-network \
  -v hjtpx-logs:/var/log/hjtpx \
  -v ./backend/config:/app/config:ro \
  hjtpx:latest
```

### 生产环境配置

#### 资源限制配置

```yaml
# docker-compose.prod.yml
services:
  app:
    image: ghcr.io/opphk/hjtpx:latest
    deploy:
      resources:
        limits:
          cpus: '2.0'
          memory: 2G
        reservations:
          cpus: '1.0'
          memory: 1G
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
```

#### 安全配置

```dockerfile
# Dockerfile
FROM golang:1.25-alpine AS builder

WORKDIR /build

COPY go.mod go.sum ./
RUN go mod download

COPY . .

RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -o hjtpx ./backend/cmd/api/main.go

FROM alpine:3.19

RUN apk --no-cache add ca-certificates tzdata && \
    adduser -D -g '' appuser

WORKDIR /app

COPY --from=builder /build/hjtpx .
COPY backend/config/config.yaml /app/config/
COPY docker/entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

USER appuser

EXPOSE 8080

HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:8080/health || exit 1

ENTRYPOINT ["/entrypoint.sh"]
```

### 多阶段构建

```dockerfile
# 第一阶段：编译
FROM golang:1.25-alpine AS builder

WORKDIR /app

COPY go.mod go.sum ./
RUN go mod download

COPY . .

RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
    go build -ldflags="-linkmode=external -extldflags=-static -w -s" \
    -o hjtpx ./backend/cmd/api/main.go

# 第二阶段：运行
FROM alpine:3.19

RUN apk --no-cache add ca-certificates tzdata curl && \
    adduser -D -g '' appuser

WORKDIR /app

COPY --from=builder /app/hjtpx .
COPY --from=builder /app/backend/config ./config

USER appuser

EXPOSE 8080

ENTRYPOINT ["./hjtpx"]
```

**优化效果**：
- 镜像体积：从 ~800MB 减小到 ~20MB
- 构建时间：增加约30秒
- 安全性提升：不包含编译器工具链

### 镜像优化

#### 镜像大小对比

| 基础镜像 | 优化前 | 优化后 | 减小比例 |
|---------|--------|--------|---------|
| golang:alpine | ~800MB | - | - |
| alpine:3.19 | - | ~20MB | ~97% |
| scratch | - | ~15MB | ~98% |

#### 构建缓存优化

```dockerfile
FROM golang:1.25-alpine AS builder

WORKDIR /app

COPY go.mod go.sum ./
RUN go mod download

COPY . .

RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -o hjtpx .
```

#### 多架构构建

```bash
docker buildx create --use
docker buildx inspect --bootstrap

docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --tag ghcr.io/opphk/hjtpx:latest \
  --push .
```

```bash

docker-compose ps

# 查看日志
docker-compose logs -f

# 进入容器
```

## Kubernetes 部署

### 前提条件

- Kubernetes 1.24+ 集群
- kubectl 配置完成
- 可选：Helm 3.0+

### 部署步骤

#### 1. 创建命名空间

```bash
kubectl create namespace hjtpx
```

#### 2. 创建 Secrets

```yaml
# hjtpx-secrets.yaml
apiVersion: v1
kind: Secret
metadata:
  name: hjtpx-secrets
  namespace: hjtpx
type: Opaque
stringData:
  postgres-password: "your-secure-password"
  redis-password: "your-redis-password"
  jwt-secret: "your-very-secure-jwt-secret-key-min-32-chars"
  grafana-password: "your-grafana-password"
```

```bash
kubectl apply -f hjtpx-secrets.yaml
```

#### 3. 部署 PostgreSQL

```yaml
# postgres.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-pvc
  namespace: hjtpx
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi

---

apiVersion: apps/v1
kind: Deployment
metadata:
  name: postgres
  namespace: hjtpx
spec:
  replicas: 1
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: postgres:16-alpine
        env:
        - name: POSTGRES_USER
          value: postgres
        - name: POSTGRES_PASSWORD
          valueFrom:
            secretKeyRef:
              name: hjtpx-secrets
              key: postgres-password
        - name: POSTGRES_DB
          value: hjtpx_db
        ports:
        - containerPort: 5432
        volumeMounts:
        - name: postgres-storage
          mountPath: /var/lib/postgresql/data
        resources:
          requests:
            cpu: "500m"
            memory: "512Mi"
          limits:
            cpu: "1500m"
            memory: "1Gi"
        livenessProbe:
          exec:
            command: ["pg_isready", "-U", "postgres"]
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          exec:
            command: ["pg_isready", "-U", "postgres"]
          initialDelaySeconds: 5
          periodSeconds: 5
      volumes:
      - name: postgres-storage
        persistentVolumeClaim:
          claimName: postgres-pvc

---

apiVersion: v1
kind: Service
metadata:
  name: postgres
  namespace: hjtpx
spec:
  selector:
    app: postgres
  ports:
  - port: 5432
    targetPort: 5432
```

```bash
kubectl apply -f postgres.yaml
```

## 手动部署

### 步骤 1：安装依赖

#### Ubuntu/Debian

```bash
sudo apt-get update
sudo apt-get install -y curl wget git make gcc

wget https://go.dev/dl/go1.25.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.25.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

sudo apt-get install -y postgresql postgresql-contrib redis-server nginx
```

#### CentOS/RHEL

```bash
sudo yum install -y epel-release
sudo yum install -y curl wget git make gcc

wget https://go.dev/dl/go1.25.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.25.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

sudo yum install -y postgresql-server postgresql-contrib redis
sudo postgresql-setup --initdb
sudo systemctl start postgresql redis
sudo systemctl enable postgresql redis
```

### 步骤 2：配置 PostgreSQL

```bash
sudo -u postgres psql << EOF
CREATE DATABASE hjtpx_db;
CREATE USER hjtpx WITH PASSWORD 'your-secure-password';
GRANT ALL PRIVILEGES ON DATABASE hjtpx_db TO hjtpx;
ALTER DATABASE hjtpx_db OWNER TO hjtpx;
\c hjtpx_db
GRANT ALL ON SCHEMA public TO hjtpx;
EOF

sudo vim /var/lib/postgresql/16/main/pg_hba.conf
sudo systemctl restart postgresql
```

### 步骤 3：配置 Redis

```bash
sudo vim /etc/redis/redis.conf

requirepass your-redis-password
appendonly yes
appendfsync everysec
maxmemory 256mb
maxmemory-policy allkeys-lru

sudo systemctl restart redis
```

### 步骤 4：获取并编译代码

```bash
git clone https://github.com/opphk/hjtpx.git
cd hjtpx/backend
go mod download
go build -o hjtpx ./cmd/api/main.go

sudo mkdir -p /var/log/hjtpx
sudo chown $USER:$USER /var/log/hjtpx
```

### 步骤 5：创建 systemd 服务

```bash
sudo vim /etc/systemd/system/hjtpx.service
```

```ini
[Unit]
Description=HJTPX Verification Service
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/hjtpx
ExecStart=/opt/hjtpx/hjtpx
Restart=always
RestartSec=5
Environment="GIN_MODE=release"
```

**A:** 调整连接池大小，启用缓存，考虑水平扩展。

## 安全加固

### HTTPS配置

```nginx
server {
    listen 80;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    ssl_certificate /etc/ssl/certs/hjtpx.crt;
    ssl_certificate_key /etc/ssl/private/hjtpx.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";
}
```

### 防火墙配置

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 8080/tcp
sudo ufw enable
```
