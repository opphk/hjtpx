## Docker部署（推荐）

### 快速部署

1. **克隆项目**

```bash
git clone https://github.com/opphk/behavior-verification-system.git
cd behavior-verification-system
```

2. **配置环境变量**

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```bash
# 数据库配置（必须修改）
POSTGRES_PASSWORD=your-secure-password
POSTGRES_USER=postgres
POSTGRES_DB=hjtpx_db

# Redis配置
REDIS_PASSWORD=your-redis-password

# JWT密钥（必须修改，至少32字符）
JWT_SECRET=your-very-secure-jwt-secret-key-min-32-chars

# Grafana密码
GRAFANA_ADMIN_PASSWORD=your-grafana-password
```

3. **启动服务**

```bash
docker-compose up -d
```

4. **验证服务状态**

```bash
docker-compose ps
```

5. **查看日志**

```bash
docker-compose logs -f app
```

## 本地部署

### 前置条件

- Go 1.25+
- PostgreSQL 12+
- Redis 6+

### 1. 数据库配置

```bash
# 安装PostgreSQL（Ubuntu/Debian）
sudo apt install postgresql postgresql-contrib

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 创建数据库
sudo -u postgres psql

CREATE DATABASE hjtpx_db;
CREATE USER postgres WITH PASSWORD 'your-password';
GRANT ALL PRIVILEGES ON DATABASE hjtpx_db TO postgres;
\q
```

### 2. Redis配置

```bash
# 安装Redis（Ubuntu/Debian）
sudo apt install redis-server

# 启动服务
sudo systemctl start redis-server
sudo systemctl enable redis-server
```

### 3. 应用配置

```bash
# 克隆项目
git clone https://github.com/opphk/behavior-verification-system.git
cd behavior-verification-system

# 安装依赖
cd backend
go mod download

# 配置文件
cp backend/config/config.yaml.example backend/config/config.yaml
```

编辑 `backend/config/config.yaml`：

```yaml
server:
  port: 8080
  mode: release

postgres:
  host: 127.0.0.1
  port: 5432
  user: postgres
  password: your-password
  db_name: hjtpx_db
  sslmode: disable

redis:
  host: 127.0.0.1
  port: 6379
  password: ""
  db: 0

jwt:
  secret: your-very-secure-jwt-secret-key-min-32-chars
  expire_hours: 24
```

### 4. 初始化数据库

```bash
psql -U postgres -d hjtpx_db -f scripts/init-db.sql
```

### 5. 运行服务

```bash
# 开发模式
cd backend
go run ./cmd/api/main.go

# 或编译后运行
go build -o hjtpx ./cmd/api/main.go
./hjtpx
```

## Nginx配置

### 基本配置

编辑 `nginx/nginx.conf`：

```nginx
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript 
               application/json application/javascript application/xml+rss;

    include /etc/nginx/conf.d/*.conf;
}
```

### 站点配置

编辑 `nginx/conf.d/default.conf`：

```nginx
upstream backend {
    server app:8080;
    keepalive 32;
}

server {
    listen 80;
    server_name localhost;

    client_max_body_size 10M;

    location /health {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
    }

    location /api/ {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    location /admin {
        alias /usr/share/nginx/html/admin;
        try_files $uri $uri/ /admin/index.html;
    }

    location / {
        root /usr/share/nginx/html/frontend;
        try_files $uri $uri/ /index.html;
    }

    location ~ /\. {
        deny all;
    }
}
```

### 性能优化

```nginx
# 缓存静态资源
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}

# 限制请求大小
client_max_body_size 10M;
client_body_buffer_size 1M;

# 连接限制
limit_conn_zone $binary_remote_addr zone=addr:10m;
location /api/ {
    limit_conn addr 10;
}
```

## HTTPS配置

### 生成SSL证书（自签名，用于测试）

```bash
# 创建证书目录
mkdir -p nginx/ssl

# 生成私钥
openssl genrsa -out nginx/ssl/server.key 2048

# 生成证书
openssl req -new -x509 -key nginx/ssl/server.key \
    -out nginx/ssl/server.crt -days 365 \
    -subj "/C=CN/ST=Beijing/L=Beijing/O=Example/CN=localhost"
```

### 生产环境配置（使用Let's Encrypt）

```bash
# 安装certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d yourdomain.com
```

### HTTPS站点配置

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    ssl_certificate /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # HSTS
    add_header Strict-Transport-Security "max-age=31536000" always;

    location /api/ {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }

    # 其他location配置...
}
```

## 监控配置

### Prometheus配置

编辑 `monitoring/prometheus/prometheus.yml`：

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'hjtpx'
    static_configs:
      - targets: ['app:8080']
    metrics_path: '/metrics'

  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']
```

### Grafana配置

Grafana已预配置数据源和仪表盘：

- **数据源**：
  - Prometheus: http://prometheus:9090
  - Loki: http://loki:3100

- **仪表盘**：
  - hjtpx-dashboard: 验证系统监控面板

### 查看监控数据

1. 访问 Grafana：http://localhost:3000
2. 登录（默认 admin/admin123）
3. 进入 Dashboard 查看监控数据

### Loki日志配置

编辑 `monitoring/loki/loki.yml`：

```yaml
auth_enabled: false

server:
  http_listen_port: 3100
  grpc_listen_port: 9096

common:
  path_prefix: /loki
  storage:
    filesystem:
      chunks_directory: /loki/chunks
      rules_directory: /loki/rules
  replication_factor: 1
  retain_period: 744h

schema_config:
  configs:
    - from: 2024-01-01
      store: boltdb
      object_store: filesystem
      schema: v11
      index:
        prefix: index_
        period: 168h

limits_config:
  reject_old_samples: true
  reject_old_samples_max_age: 168h
```

### Promtail配置

编辑 `monitoring/promtail/promtail.yml`：

```yaml
server:
  http_listen_port: 9080
  grpc_listen_port: 0

positions:
  filename: /var/log/positions.yaml

clients:
  - url: http://loki:3100/loki/api/v1/push

scrape_configs:
  - job_name: hjtpx
    static_configs:
      - targets:
          - localhost
        labels:
          job: hjtpx
          __path__: /var/log/hjtpx/*.log

  - job_name: nginx
    static_configs:
      - targets:
          - localhost
        labels:
          job: nginx
          __path__: /var/log/nginx/*.log
```

## 故障排除

### 常见问题

#### 1. 服务无法启动

```bash
# 检查端口占用
netstat -tlnp | grep 8080

# 检查Docker状态
docker-compose ps

# 查看详细日志
docker-compose logs -f
```

#### 2. 数据库连接失败

```bash
# 检查PostgreSQL状态
docker-compose ps postgres

# 测试连接
docker-compose exec postgres psql -U postgres -d hjtpx_db

# 检查连接配置
docker-compose exec app sh -c 'echo $POSTGRES_HOST'
```

#### 3. Redis连接失败

```bash
# 检查Redis状态
docker-compose ps redis

# 测试连接
docker-compose exec redis redis-cli ping
```

#### 4. 验证码无法生成

```bash
# 检查会话存储
docker-compose exec app sh -c 'redis-cli -a password GET sess_*'

# 检查日志
docker-compose logs app | grep -i error
```

#### 5. 性能问题

```bash
# 查看资源使用
docker stats

# 检查慢查询
docker-compose logs app | grep -i slow

# Prometheus查看QPS
curl http://localhost:9090/api/v1/query?query=http_requests_total
```

## 安全建议

### 生产环境必做

1. **修改所有默认密码**
2. **启用HTTPS**
3. **配置防火墙**
4. **启用日志审计**
5. **定期备份数据**
6. **监控安全事件**
7. **使用SSL/TLS加密通信**
8. **配置IP白名单**
9. **启用CSRF/XSS防护**
10. **定期更新依赖**

### 防火墙配置

```bash
# Ubuntu/Debian
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable
```

```bash
# 安装
sudo apt install fail2ban

logpath = /var/log/nginx/error.log
maxretry = 5
```
