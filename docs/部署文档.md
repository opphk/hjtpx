## Docker部署（推荐）

### 快速部署

1. **克隆项目**

```bash
git clone https://github.com/opphk/behavior-verification-system.git
cd behavior-verification-system
```

2. **配置环境变量**

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```bash
# 数据库配置（必须修改）
POSTGRES_PASSWORD=your-secure-password
POSTGRES_USER=postgres
POSTGRES_DB=hjtpx_db

# Redis配置
REDIS_PASSWORD=your-redis-password

# JWT密钥（必须修改，至少32字符）
JWT_SECRET=your-very-secure-jwt-secret-key-min-32-chars

# Grafana密码
GRAFANA_ADMIN_PASSWORD=your-grafana-password

# 资源限制配置
APP_CPU_LIMIT=1.0
APP_MEMORY_LIMIT=512M
POSTGRES_CPU_LIMIT=1.5
POSTGRES_MEMORY_LIMIT=1G
REDIS_CPU_LIMIT=0.5
REDIS_MEMORY_LIMIT=512M
```

3. **创建SSL证书目录**

```bash
mkdir -p nginx/ssl
```

生成自签名证书或放置真实证书：

```bash
# 自签名证书（仅用于测试）
./scripts/ssl-renew.sh
```

4. **启动服务**

```bash
docker-compose up -d
```

5. **验证服务状态**

```bash
docker-compose ps
```

6. **检查健康状态**

```bash
./scripts/health-check.sh
```

## 本地部署

### 前置条件

- Go 1.25+
- PostgreSQL 12+
- Redis 6+

### 1. 数据库配置

```bash
# 安装PostgreSQL（Ubuntu/Debian）
sudo apt install postgresql postgresql-contrib

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 创建数据库
sudo -u postgres psql

CREATE DATABASE hjtpx_db;
CREATE USER postgres WITH PASSWORD 'your-password';
GRANT ALL PRIVILEGES ON DATABASE hjtpx_db TO postgres;
\q
```

### 2. Redis配置

```bash
# 安装Redis（Ubuntu/Debian）
sudo apt install redis-server

# 启动服务
sudo systemctl start redis-server
sudo systemctl enable redis-server
```

### 3. 应用配置

```bash
# 克隆项目
git clone https://github.com/opphk/behavior-verification-system.git
cd behavior-verification-system

# 安装依赖
cd backend
go mod download

# 配置文件
cp backend/config/config.yaml.example backend/config/config.yaml
```

编辑 `backend/config/config.yaml`：

```yaml
server:
  port: 8080
  mode: release

postgres:
  host: 127.0.0.1
  port: 5432
  user: postgres
  password: your-password
  db_name: hjtpx_db
  sslmode: disable

redis:
  host: 127.0.0.1
  port: 6379
  password: ""
  db: 0

jwt:
  secret: your-very-secure-jwt-secret-key-min-32-chars
  expire_hours: 24
```

### 4. 初始化数据库

```bash
psql -U postgres -d hjtpx_db -f scripts/init-db.sql
```

### 5. 运行服务

```bash
# 开发模式
cd backend
go run ./cmd/api/main.go

# 或编译后运行
go build -o hjtpx ./cmd/api/main.go
./hjtpx
```

## Nginx配置

### 多阶段构建Dockerfile优化

项目使用多阶段构建优化镜像大小：

```dockerfile
# 构建阶段
FROM golang:1.25-alpine AS builder
RUN apk add --no-cache --virtual .build-deps git ca-certificates tzdata
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -trimpath -o hjtpx ./backend/cmd/api/main.go

# 运行阶段
FROM alpine:3.19
RUN apk add --no-cache ca-certificates tzdata
RUN addgroup -g 1000 appgroup && adduser -u 1000 -G appgroup -s /bin/sh -D appuser
COPY --from=builder /build/hjtpx .
USER appuser
```

### 基本配置

编辑 `nginx/nginx.conf`：

```nginx
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;
worker_rlimit_nofile 65535;

events {
    worker_connections 2048;
    use epoll;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format json_log escape=json '{'
        '"time":"$time_iso8601",'
        '"remote_addr":"$remote_addr",'
        '"request":"$request",'
        '"status":$status,'
        '"request_time":$request_time'
    '}';

    access_log /var/log/nginx/access.log json_log;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    keepalive_requests 10000;

    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript;

    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=100r/s;
    limit_conn_zone $binary_remote_addr zone=addr:10m;

    include /etc/nginx/conf.d/*.conf;
}
```

### 站点配置

编辑 `nginx/conf.d/default.conf`：

```nginx
upstream hjtpx_backend {
    least_conn;
    server app:8080 max_fails=3 fail_timeout=30s;
    keepalive 32;
}

server {
    listen 80;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name localhost;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    location /api/ {
        limit_req zone=api_limit burst=200 nodelay;
        limit_conn addr 20;
        proxy_pass http://hjtpx_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
```

### 性能优化

```nginx
# 文件缓存
open_file_cache max=10000 inactive=30s;
open_file_cache_valid 60s;
open_file_cache_min_uses 2;

# 客户端优化
client_body_timeout 12s;
client_header_timeout 12s;
client_max_body_size 10M;

# 代理优化
proxy_buffer_size 128k;
proxy_buffers 4 256k;
proxy_connect_timeout 60s;
```

## HTTPS配置

### 生成SSL证书（自签名，用于测试）

```bash
mkdir -p nginx/ssl
./scripts/ssl-renew.sh
```

### 生产环境配置（使用Let's Encrypt）

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com
```

### HTTPS站点配置

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    ssl_certificate /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    add_header Strict-Transport-Security "max-age=31536000" always;

    location /api/ {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}
```

### SSL证书自动续期

使用脚本自动续期证书：

```bash
# 配置环境变量
export CERT_EMAIL="admin@example.com"
export DOMAINS="yourdomain.com"

# 运行续期脚本
./scripts/ssl-renew.sh
```

```bash
gunzip < backup_20240516.sql.gz | psql -U postgres -d hjtpx_db
```

## 故障排除

```bash
docker-compose ps
```

```bash
ls -la nginx/ssl/
```

## 安全建议

### 生产环境必做

1. **修改所有默认密码**
2. **启用HTTPS**
3. **配置防火墙**
4. **启用日志审计**
5. **定期备份数据**
6. **监控安全事件**
7. **使用SSL/TLS加密通信**
8. **配置IP白名单**
9. **启用CSRF/XSS防护**
10. **定期更新依赖**

```bash
sudo ufw allow 80/tcp
```
