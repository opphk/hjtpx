## 环境要求

### 硬件要求

| 部署规模 | CPU | 内存 | 磁盘 | 网络 |
|---------|-----|------|------|------|
| 开发/测试 | 2核 | 4GB | 20GB | 100Mbps |
| 小型生产（<100 QPS） | 4核 | 8GB | 50GB | 1Gbps |
| 中型生产（100-1000 QPS） | 8核 | 16GB | 100GB | 1Gbps |
| 大型生产（>1000 QPS） | 16核+ | 32GB+ | 500GB+ | 10Gbps |

### 软件要求

| 组件 | 最低版本 | 推荐版本 | 说明 |
|------|---------|---------|------|
| Go | 1.21+ | 1.25+ | 仅手动部署需要 |
| PostgreSQL | 12+ | 16+ | 关系型数据库 |
| Redis | 6+ | 7+ | 缓存和会话存储 |
| Nginx | 1.18+ | 1.25+ | 反向代理（可选） |
| Docker | 20.10+ | 24.0+ | 容器化部署 |
| Kubernetes | 1.24+ | 1.28+ | 容器编排（可选） |

### 操作系统支持

- Linux: Ubuntu 20.04+, Debian 11+, CentOS 8+, RHEL 8+
- macOS: 11+ (仅限开发环境)
- Windows: 10+ (仅限开发环境，推荐使用 WSL2)

## 快速部署

### 使用 Docker Compose（推荐）

```bash
# 1. 克隆仓库
git clone https://github.com/hjtpx/hjtpx.git
cd hjtpx

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，修改密码等敏感配置
vim .env

# 3. 启动所有服务
docker-compose up -d

# 4. 查看服务状态
docker-compose ps

# 5. 查看日志
docker-compose logs -f
```

## Docker 部署

### 方式一：使用官方镜像（待发布）

```bash
# 拉取镜像
docker pull hjtpx/hjtpx:latest

# 运行容器
docker run -d \
  --name hjtpx-app \
  -p 8080:8080 \
  -e POSTGRES_HOST=your-postgres-host \
  -e POSTGRES_PORT=5432 \
  -e POSTGRES_DB=hjtpx_db \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=your-password \
  -e REDIS_HOST=your-redis-host \
  -e REDIS_PORT=6379 \
  -e REDIS_PASSWORD=your-redis-password \
  -e JWT_SECRET=your-very-secure-jwt-secret-key \
  hjtpx/hjtpx:latest
```

### 方式二：自行构建镜像

```bash
# 克隆代码
git clone https://github.com/hjtpx/hjtpx.git
cd hjtpx

# 构建镜像
docker build -t hjtpx:latest .

# 运行容器
docker run -d \
  --name hjtpx-app \
  -p 8080:8080 \
  --network hjtpx-network \
  -v hjtpx-logs:/var/log/hjtpx \
  -v ./backend/config:/app/config:ro \
  hjtpx:latest
```

### Docker 运行参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `-d` | 后台运行 | `-d` |
| `--name` | 容器名称 | `--name hjtpx-app` |
| `-p` | 端口映射 | `-p 8080:8080` |
| `-e` | 环境变量 | `-e POSTGRES_HOST=postgres` |
| `-v` | 卷挂载 | `-v app-logs:/var/log/hjtpx` |
| `--network` | 网络 | `--network hjtpx-network` |
| `--restart` | 重启策略 | `--restart unless-stopped` |

### 完整的 docker-compose.yml 说明

项目根目录已提供完整的 `docker-compose.yml`，包含以下服务：

#### 服务架构图

```
                    ┌─────────────┐
                    │   Nginx     │
                    │  (反向代理)  │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────▼────┐       ┌─────▼─────┐      ┌────▼────┐
   │  App    │       │ Prometheus│      │ Grafana│
   │ (Go API)│       │  (指标)    │      │ (可视化)│
   └────┬────┘       └───────────┘      └─────────┘
        │
   ┌────┴────┐
```

### 环境变量配置

复制 `.env.example` 为 `.env` 并根据需要修改：

```bash
# 应用配置
GIN_MODE=release                    # 运行模式: debug/release
SERVER_PORT=8080                    # 服务端口

# PostgreSQL 配置
POSTGRES_HOST=postgres              # 数据库主机
POSTGRES_PORT=5432                  # 数据库端口
POSTGRES_USER=postgres              # 数据库用户
POSTGRES_PASSWORD=your-secure-password  # 数据库密码
POSTGRES_DB=hjtpx_db                # 数据库名
POSTGRES_SSLMODE=disable            # SSL 模式

# Redis 配置
REDIS_HOST=redis                    # Redis 主机
REDIS_PORT=6379                     # Redis 端口
REDIS_PASSWORD=your-redis-password  # Redis 密码
REDIS_MAXMEMORY=256mb               # Redis 最大内存

# JWT 配置
JWT_SECRET=your-very-secure-jwt-secret-key-min-32-chars  # JWT 密钥
JWT_EXPIRE_HOURS=24                 # Token 过期时间（小时）

# 监控配置
PROMETHEUS_PORT=9090                # Prometheus 端口
GRAFANA_PORT=3000                   # Grafana 端口
LOKI_PORT=3100                      # Loki 端口
GRAFANA_ADMIN_USER=admin            # Grafana 管理员用户
GRAFANA_ADMIN_PASSWORD=admin123     # Grafana 管理员密码
```

### 常用 Docker Compose 命令

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启服务
docker-compose restart

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
docker-compose logs -f app
docker-compose logs --tail 100 app

# 进入容器
docker-compose exec app sh
docker-compose exec postgres psql -U postgres -d hjtpx_db
docker-compose exec redis redis-cli

# 重新构建并启动
docker-compose up -d --build

# 更新镜像并重启
docker-compose pull
docker-compose up -d

# 清理未使用的资源
docker-compose down -v
docker system prune -a
```

## Kubernetes 部署

### 前提条件

- Kubernetes 1.24+ 集群
- kubectl 配置完成
- 可选：Helm 3.0+

### 部署步骤

#### 1. 创建命名空间

```bash
kubectl create namespace hjtpx
```

#### 2. 创建 Secrets

```yaml
# hjtpx-secrets.yaml
apiVersion: v1
kind: Secret
metadata:
  name: hjtpx-secrets
  namespace: hjtpx
type: Opaque
stringData:
  postgres-password: "your-secure-password"
  redis-password: "your-redis-password"
  jwt-secret: "your-very-secure-jwt-secret-key-min-32-chars"
  grafana-password: "your-grafana-password"
```

```bash
kubectl apply -f hjtpx-secrets.yaml
```

#### 3. 部署 PostgreSQL

```yaml
# postgres.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-pvc
  namespace: hjtpx
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi

---

apiVersion: apps/v1
kind: Deployment
metadata:
  name: postgres
  namespace: hjtpx
spec:
  replicas: 1
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: postgres:16-alpine
        env:
        - name: POSTGRES_USER
          value: postgres
        - name: POSTGRES_PASSWORD
          valueFrom:
            secretKeyRef:
              name: hjtpx-secrets
              key: postgres-password
        - name: POSTGRES_DB
          value: hjtpx_db
        - name: PGDATA
          value: /var/lib/postgresql/data/pgdata
        ports:
        - containerPort: 5432
        volumeMounts:
        - name: postgres-storage
          mountPath: /var/lib/postgresql/data
        resources:
          requests:
            cpu: "500m"
            memory: "512Mi"
          limits:
            cpu: "1500m"
            memory: "1Gi"
        livenessProbe:
          exec:
            command:
            - pg_isready
            - -U
            - postgres
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          exec:
            command:
            - pg_isready
            - -U
            - postgres
          initialDelaySeconds: 5
          periodSeconds: 5
      volumes:
      - name: postgres-storage
        persistentVolumeClaim:
          claimName: postgres-pvc

---

apiVersion: v1
kind: Service
metadata:
  name: postgres
  namespace: hjtpx
spec:
  selector:
    app: postgres
  ports:
  - port: 5432
    targetPort: 5432
```

```bash
kubectl apply -f postgres.yaml
```

#### 4. 部署 Redis

```yaml
# redis.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: redis-pvc
  namespace: hjtpx

  accessModes:
    - ReadWriteOnce

  name: redis
  namespace: hjtpx

  replicas: 1

      labels:
        app: redis

            cpu: "500m"
            memory: "512Mi"

            - redis-cli
            - --no-auth-warning
            - -a
            - $(REDIS_PASSWORD)

            - redis-cli
            - --no-auth-warning
```

#### 5. 部署应用

```yaml
# app.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: hjtpx-config
  namespace: hjtpx
data:

  SERVER_PORT: "8080"

---

apiVersion: apps/v1
kind: Deployment
metadata:
  name: hjtpx-app
  namespace: hjtpx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: hjtpx-app

    metadata:
      labels:
        app: hjtpx-app

          valueFrom:
            secretKeyRef:
              name: hjtpx-secrets

        - name: JWT_SECRET
          valueFrom:
            secretKeyRef:
              name: hjtpx-secrets

        ports:
        - containerPort: 8080
          name: http
        resources:
          requests:
            cpu: "500m"
            memory: "256Mi"
          limits:
            cpu: "1000m"
            memory: "512Mi"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 40
          periodSeconds: 30
          timeoutSeconds: 10

          httpGet:
            path: /health
            port: 8080

  name: hjtpx-app
  namespace: hjtpx
spec:

    app: hjtpx-app

apiVersion: autoscaling/v2

  name: hjtpx-app-hpa

  scaleTargetRef:

  maxReplicas: 10
  metrics:
  - type: Resource

      name: cpu

        type: Utilization
```

## 手动部署

### 步骤 1：安装依赖

#### Ubuntu/Debian

```bash
# 更新包列表
sudo apt-get update

# 安装基础工具
sudo apt-get install -y curl wget git make gcc

# 安装 Go 1.25
wget https://go.dev/dl/go1.25.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.25.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

# 安装 PostgreSQL
sudo apt-get install -y postgresql postgresql-contrib

# 安装 Redis
sudo apt-get install -y redis-server

# 安装 Nginx（可选）
sudo apt-get install -y nginx
```

#### CentOS/RHEL

```bash
# 安装 EPEL
sudo yum install -y epel-release

# 安装基础工具
sudo yum install -y curl wget git make gcc

# 安装 Go 1.25
wget https://go.dev/dl/go1.25.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.25.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

# 安装 PostgreSQL
sudo yum install -y postgresql-server postgresql-contrib
sudo postgresql-setup --initdb
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 安装 Redis
sudo yum install -y redis
sudo systemctl start redis
sudo systemctl enable redis
```

### 步骤 2：配置 PostgreSQL

```bash
# 切换到 postgres 用户
sudo -u postgres psql << EOF
-- 创建数据库
CREATE DATABASE hjtpx_db;

-- 创建用户
CREATE USER hjtpx WITH PASSWORD 'your-secure-password';

-- 授权
GRANT ALL PRIVILEGES ON DATABASE hjtpx_db TO hjtpx;
ALTER DATABASE hjtpx_db OWNER TO hjtpx;

-- 连接到数据库并设置权限
\c hjtpx_db
GRANT ALL ON SCHEMA public TO hjtpx;
EOF

# 修改 pg_hba.conf 允许密码认证
sudo vim /var/lib/postgresql/16/main/pg_hba.conf
# 将 local 和 host 的 peer/ident 改为 md5

# 重启 PostgreSQL
sudo systemctl restart postgresql
```

### 步骤 3：配置 Redis

```bash
# 编辑 Redis 配置
sudo vim /etc/redis/redis.conf

# 设置密码
# requirepass your-redis-password

# 配置持久化
appendonly yes
appendfsync everysec

# 配置内存策略
maxmemory 256mb
maxmemory-policy allkeys-lru

# 重启 Redis
sudo systemctl restart redis
```

### 步骤 4：获取并编译代码

```bash
# 克隆仓库
git clone https://github.com/hjtpx/hjtpx.git
cd hjtpx

# 进入后端目录
cd backend

# 下载依赖
go mod download

# 编译
go build -o hjtpx ./cmd/api/main.go

# 创建日志目录
sudo mkdir -p /var/log/hjtpx
sudo chown $USER:$USER /var/log/hjtpx

# 复制配置文件
cp config/config.yaml /etc/hjtpx/config.yaml
```

### 步骤 5：配置应用

编辑 `/etc/hjtpx/config.yaml`：

```yaml
server:
  port: 8080
  mode: release

postgres:
  host: localhost
  port: 5432
  user: hjtpx
  password: your-secure-password
  db_name: hjtpx_db
  sslmode: disable
  max_open_conns: 100
  max_idle_conns: 20
  conn_max_lifetime: 300

redis:
  host: localhost
  port: 6379
  password: your-redis-password
  db: 0
  max_retries: 3

jwt:
  secret: your-very-secure-jwt-secret-key-min-32-chars
  expire_hours: 24
```

### 步骤 6：初始化数据库

```bash
# 执行初始化脚本
psql -U hjtpx -d hjtpx_db -f scripts/init-db.sql
```

### 步骤 7：创建 systemd 服务

```bash
sudo vim /etc/systemd/system/hjtpx.service
```

```ini
[Unit]
Description=HJTPX Verification Service
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/hjtpx
ExecStart=/opt/hjtpx/hjtpx
Restart=always
RestartSec=5
Environment="GIN_MODE=release"
Environment="SERVER_PORT=8080"
Environment="POSTGRES_HOST=localhost"
Environment="POSTGRES_PORT=5432"
Environment="POSTGRES_USER=hjtpx"
Environment="POSTGRES_PASSWORD=your-secure-password"
Environment="POSTGRES_DB=hjtpx_db"
Environment="REDIS_HOST=localhost"
Environment="REDIS_PORT=6379"
Environment="JWT_SECRET=your-very-secure-jwt-secret-key-min-32-chars"

# 安全设置
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/var/log/hjtpx

[Install]
WantedBy=multi-user.target
```

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start hjtpx

# 设置开机自启
sudo systemctl enable hjtpx

# 查看状态
sudo systemctl status hjtpx

# 查看日志
sudo journalctl -u hjtpx -f
```

### 步骤 8：配置 Nginx（可选）

```bash
sudo vim /etc/nginx/sites-available/hjtpx
```

```nginx
upstream hjtpx_backend {
    server 127.0.0.1:8080;
    keepalive 32;
}

server {
    listen 80;
    server_name your-domain.com;

    # 日志
    access_log /var/log/nginx/hjtpx_access.log;
    error_log /var/log/nginx/hjtpx_error.log;

    # 客户端限制
    client_max_body_size 10M;
    client_body_timeout 60s;

    location / {
        proxy_pass http://hjtpx_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 30s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
        
        # 缓冲设置
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 8 4k;
    }

    # 静态文件缓存
    location /static/ {
        alias /opt/hjtpx/frontend/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    location /admin/static/ {
        alias /opt/hjtpx/admin/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 健康检查
    location /health {
        proxy_pass http://hjtpx_backend;
        access_log off;
    }
}
```

```bash
# 启用站点
sudo ln -s /etc/nginx/sites-available/hjtpx /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 常见问题

### Q: 容器启动失败怎么办？

A: 检查日志：
```bash
docker-compose logs app
```
常见原因：
- 数据库连接失败：检查 PostgreSQL 是否启动
- Redis 连接失败：检查 Redis 是否启动
- 端口被占用：检查 8080 端口是否被占用

### Q: 如何修改默认密码？

A: 
- 管理后台：登录后在设置中修改
- Grafana：登录后在 Configuration → Preferences 中修改
- 数据库：修改 .env 文件中的 POSTGRES_PASSWORD 并重启

### Q: 数据如何备份？

A: 参考 [运维手册](./运维手册.md) 中的备份恢复章节。

A: 
```bash
# Docker Compose
git pull
docker-compose down
docker-compose build
docker-compose up -d

# Kubernetes
kubectl set image deployment/hjtpx-app hjtpx-app=hjtpx/hjtpx:new-version
```
