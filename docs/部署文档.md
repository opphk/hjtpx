## 快速部署

### 使用 Docker Compose

```bash
# 克隆项目
```

## Docker 部署

### 构建镜像

```bash
docker build -t hjtpx:latest .
```

### 运行容器

```bash
docker run -d \
  --name hjtpx \
  -p 8080:8080 \
  -e POSTGRES_HOST=postgres \
  -e POSTGRES_PORT=5432 \
  -e POSTGRES_DB=hjtpx_db \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=your-password \
  -e REDIS_HOST=redis \
  -e REDIS_PORT=6379 \
  -e JWT_SECRET=your-jwt-secret \
  hjtpx:latest
```

### Docker Compose 服务说明

| 服务 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| app | 本地构建 | 8080 | Go API服务 |
| postgres | postgres:16-alpine | 5432 | PostgreSQL数据库 |
| redis | redis:7-alpine | 6379 | Redis缓存 |
| nginx | nginx:1.25-alpine | 80, 443 | 反向代理 |
| prometheus | prom/prometheus | 9090 | 指标收集 |
| grafana | grafana/grafana | 3000 | 监控面板 |

## 手动部署

### 1. 安装依赖

```bash
# 安装 Go
wget https://go.dev/dl/go1.25.linux-amd64.tar.gz
tar -C /usr/local -xzf go1.25.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

# 安装 PostgreSQL
apt-get update
apt-get install -y postgresql postgresql-contrib

# 安装 Redis
apt-get install -y redis-server

# 启动服务
systemctl start postgresql
systemctl start redis-server
systemctl enable postgresql
systemctl enable redis-server
```

### 2. 配置数据库

```bash
# 创建数据库和用户
sudo -u postgres psql << EOF
CREATE DATABASE hjtpx_db;
CREATE USER hjtpx WITH PASSWORD 'your-password';
GRANT ALL PRIVILEGES ON DATABASE hjtpx_db TO hjtpx;
ALTER DATABASE hjtpx_db OWNER TO hjtpx;
\q
EOF
```

### 3. 编译项目

```bash
cd backend
go mod download
go build -o hjtpx ./cmd/api/main.go
```

### 4. 配置 Nginx

编辑 `/etc/nginx/sites-available/hjtpx`:

```nginx
upstream hjtpx_backend {
    server 127.0.0.1:8080;
    keepalive 32;
}

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://hjtpx_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

启用配置:

```bash
ln -s /etc/nginx/sites-available/hjtpx /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

## 配置

### 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| POSTGRES_HOST | PostgreSQL 主机 | localhost |
| POSTGRES_PORT | PostgreSQL 端口 | 5432 |
| POSTGRES_DB | 数据库名 | hjtpx_db |
| POSTGRES_USER | 数据库用户 | postgres |
| POSTGRES_PASSWORD | 数据库密码 | - |
| REDIS_HOST | Redis 主机 | localhost |
| REDIS_PORT | Redis 端口 | 6379 |
| REDIS_PASSWORD | Redis 密码 | - |
| JWT_SECRET | JWT 密钥 | - |
| JWT_EXPIRE_HOURS | Token过期时间(小时) | 24 |
| SERVER_PORT | 服务端口 | 8080 |
| SERVER_MODE | 运行模式(debug/release) | release |

### 配置文件

编辑 `backend/config/config.yaml`:

```yaml
server:
  port: 8080
  mode: release

database:
  host: 127.0.0.1
  port: 5432
  user: postgres
  password: your-password
  db_name: hjtpx_db
  sslmode: disable
  max_open_conns: 100
  max_idle_conns: 10

redis:
  host: 127.0.0.1
  port: 6379
  password: ""
  db: 0
  pool_size: 100

jwt:
  secret: your-jwt-secret-key
  expire_hours: 24

captcha:
  slider:
    width: 360
    height: 220
  click:
    points: 3
  image:
    length: 4
```

## 备份

### 数据库备份

```bash
# 全量备份
pg_dump -h localhost -U postgres hjtpx_db > backup_$(date +%Y%m%d).sql

# 压缩备份
pg_dump -h localhost -U postgres hjtpx_db | gzip > backup_$(date +%Y%m%d).sql.gz

# 定时备份 (crontab)
0 2 * * * /opt/hjtpx/scripts/backup.sh
```

### Redis 备份

```bash
# 保存 RDB 文件
redis-cli SAVE

# 备份文件位置
ls -la /var/lib/redis/
```

### 备份脚本

创建 `/opt/hjtpx/scripts/backup.sh`:

```bash
#!/bin/bash
BACKUP_DIR="/opt/hjtpx/backups"
DATE=$(date +%Y%m%d)

mkdir -p $BACKUP_DIR

# 数据库备份
pg_dump -h localhost -U postgres hjtpx_db | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# Redis 备份
cp /var/lib/redis/dump.rdb $BACKUP_DIR/redis_$DATE.rdb

# 清理30天前的备份
find $BACKUP_DIR -mtime +30 -delete

echo "Backup completed: $DATE"
```

### 数据恢复

```bash
# 恢复数据库
gunzip < backup_20240516.sql.gz | psql -h localhost -U postgres -d hjtpx_db

# 恢复 Redis
cp redis_20240516.rdb /var/lib/redis/dump.rdb
systemctl restart redis
```

## 故障排除

### 服务启动失败

1. 检查日志:
```bash
docker-compose logs backend
```

2. 检查数据库连接:
```bash
docker-compose exec postgres pg_isready -U postgres
```

3. 检查 Redis 连接:
```bash
docker-compose exec redis redis-cli ping
```

4. 检查端口占用:
```bash
netstat -tlnp | grep 8080
```

### 数据库迁移失败

1. 检查数据库权限:
```bash
docker-compose exec postgres psql -U postgres -c "\l"
```

2. 手动执行迁移:
```bash
psql -h localhost -U postgres -d hjtpx_db -f scripts/init-db.sql
```

3. 查看迁移日志:
```bash
docker-compose logs backend | grep -i migrate
```

### 性能问题

1. 查看资源使用:
```bash
docker stats
```

2. 查看慢查询:
```bash
docker-compose logs postgres | grep -i slow
```

3. 查看应用指标:
```bash
curl http://localhost:8080/metrics
```

### 网络问题

1. 检查容器网络:
```bash
docker network ls
docker network inspect hjtpx_default
```

2. 检查 DNS 解析:
```bash
docker-compose exec backend ping postgres
```

## 安全建议

### 生产环境必做

1. 修改所有默认密码
2. 启用 HTTPS
3. 配置防火墙
4. 启用日志审计
5. 定期备份数据
6. 监控安全事件
7. 使用 SSL/TLS 加密通信
8. 配置 IP 白名单
9. 启用 CSRF/XSS 防护
10. 定期更新依赖

### 防火墙配置

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```
