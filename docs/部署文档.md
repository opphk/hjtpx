## 环境要求

| 部署规模 | CPU | 内存 | 磁盘 | 网络 |

| Nginx | 1.18+ | 1.25+ | 反向代理（可选） |

- macOS: 11+ (仅限开发环境)
- Windows: 10+ (仅限开发环境，推荐使用 WSL2)

## Docker 部署

### 基础部署

#### 方式一：使用官方镜像

```bash
# 拉取镜像
docker pull ghcr.io/opphk/hjtpx:latest

# 运行容器
docker run -d \
  --name hjtpx-app \
  -p 8080:8080 \
  -e POSTGRES_HOST=your-postgres-host \
  -e POSTGRES_PORT=5432 \
  -e POSTGRES_DB=hjtpx_db \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=your-password \
  -e REDIS_HOST=your-redis-host \
  -e REDIS_PORT=6379 \
  -e REDIS_PASSWORD=your-redis-password \
  -e JWT_SECRET=your-very-secure-jwt-secret-key \
  ghcr.io/opphk/hjtpx:latest
```

#### 方式二：自行构建镜像

```bash
# 克隆代码
git clone https://github.com/opphk/hjtpx.git
cd hjtpx

# 构建镜像
docker build -t hjtpx:latest .

# 运行容器
docker run -d \
  --name hjtpx-app \
  -p 8080:8080 \
  --network hjtpx-network \
  -v hjtpx-logs:/var/log/hjtpx \
  -v ./backend/config:/app/config:ro \
  hjtpx:latest
```

### 生产环境配置

#### 资源限制配置

```yaml
# docker-compose.prod.yml
services:

    deploy:

      interval: 30s

      retries: 3
      start_period: 40s
```

#### 安全配置

```dockerfile
# Dockerfile
FROM golang:1.25-alpine AS builder

# 设置工作目录
WORKDIR /build

# 复制源代码
COPY . .

# 编译
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -o hjtpx ./backend/cmd/api/main.go

# 运行阶段
FROM alpine:3.19

# 安装CA证书和时区数据
RUN apk --no-cache add ca-certificates tzdata

# 创建非root用户
RUN adduser -D -g '' appuser

WORKDIR /app

# 复制二进制文件
COPY --from=builder /build/hjtpx .

# 复制配置文件
COPY backend/config/config.yaml /app/config/

# 复制脚本
COPY docker/entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

# 切换用户
USER appuser

# 暴露端口
EXPOSE 8080

# 健康检查
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:8080/health || exit 1

# 启动命令
ENTRYPOINT ["/entrypoint.sh"]
```

### 多阶段构建

使用多阶段构建可以显著减小镜像体积：

```dockerfile
# 第一阶段：编译
FROM golang:1.25-alpine AS builder

WORKDIR /app

# 安装依赖
COPY go.mod go.sum ./
RUN go mod download

# 复制源代码
COPY . .

# 编译（启用Pie和死代码消除）
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
    go build -ldflags="-linkmode=external -extldflags=-static -w -s" \
    -o hjtpx ./backend/cmd/api/main.go

# 第二阶段：运行
FROM alpine:3.19

RUN apk --no-cache add ca-certificates tzdata curl && \
    adduser -D -g '' appuser

WORKDIR /app

# 复制编译产物
COPY --from=builder /app/hjtpx .
COPY --from=builder /app/backend/config ./config

USER appuser

EXPOSE 8080

ENTRYPOINT ["./hjtpx"]
```

**优化效果**：
- 镜像体积：从 ~800MB 减小到 ~20MB
- 构建时间：增加约30秒
- 安全性提升：不包含编译器工具链

### 镜像优化

#### 镜像大小对比

| 基础镜像 | 优化前 | 优化后 | 减小比例 |
|---------|--------|--------|---------|
| golang:alpine | ~800MB | - | - |
| alpine:3.19 | - | ~20MB | ~97% |
| scratch | - | ~15MB | ~98% |

#### 构建缓存优化

```dockerfile
# 利用Docker构建缓存
FROM golang:1.25-alpine AS builder

WORKDIR /app

# 先复制依赖文件（变化少）
COPY go.mod go.sum ./
RUN go mod download

# 再复制源代码（变化频繁）
COPY . .

# 编译
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -o hjtpx .
```

#### 多架构构建

```bash
# 构建多架构镜像
docker buildx create --use
docker buildx inspect --bootstrap

# 编译多架构版本
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --tag ghcr.io/opphk/hjtpx:latest \
  --push .
```

| 参数 | 说明 | 示例 |
|------|------|------|

### 环境变量配置

复制 `.env.example` 为 `.env` 并根据需要修改：

```bash
# 应用配置
GIN_MODE=release                    # 运行模式: debug/release
SERVER_PORT=8080                    # 服务端口

# PostgreSQL 配置
POSTGRES_HOST=postgres              # 数据库主机
POSTGRES_PORT=5432                  # 数据库端口
POSTGRES_USER=postgres              # 数据库用户
POSTGRES_PASSWORD=your-secure-password  # 数据库密码
POSTGRES_DB=hjtpx_db                # 数据库名
POSTGRES_SSLMODE=disable            # SSL 模式

# Redis 配置
REDIS_HOST=redis                    # Redis 主机
REDIS_PORT=6379                     # Redis 端口
REDIS_PASSWORD=your-redis-password  # Redis 密码
REDIS_MAXMEMORY=256mb               # Redis 最大内存

# JWT 配置
JWT_SECRET=your-very-secure-jwt-secret-key-min-32-chars  # JWT 密钥
JWT_EXPIRE_HOURS=24                 # Token 过期时间（小时）

# 监控配置
PROMETHEUS_PORT=9090                # Prometheus 端口
GRAFANA_PORT=3000                   # Grafana 端口
LOKI_PORT=3100                      # Loki 端口
GRAFANA_ADMIN_USER=admin            # Grafana 管理员用户
GRAFANA_ADMIN_PASSWORD=admin123     # Grafana 管理员密码
```

### 常用 Docker Compose 命令

```bash
# 启动所有服务
docker-compose up -d

# 重启服务
docker-compose restart

docker-compose ps

# 查看日志
docker-compose logs -f

# 进入容器
docker-compose exec app sh

```

## Kubernetes 部署

### 前提条件

- Kubernetes 1.24+ 集群
- kubectl 配置完成

### 部署步骤

#### 1. 创建命名空间

```bash
kubectl create namespace hjtpx
```

#### 2. 创建 Secrets

```yaml
# hjtpx-secrets.yaml
apiVersion: v1
kind: Secret
metadata:
  name: hjtpx-secrets
  namespace: hjtpx
type: Opaque
stringData:
  postgres-password: "your-secure-password"
  redis-password: "your-redis-password"
  jwt-secret: "your-very-secure-jwt-secret-key-min-32-chars"
  grafana-password: "your-grafana-password"
```

```bash
kubectl apply -f hjtpx-secrets.yaml
```

#### 3. 部署 PostgreSQL

```yaml
# postgres.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-pvc
  namespace: hjtpx
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi

---

apiVersion: apps/v1
kind: Deployment
metadata:
  name: postgres
  namespace: hjtpx
spec:
  replicas: 1
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: postgres:16-alpine
        env:
        - name: POSTGRES_USER
          value: postgres
        - name: POSTGRES_PASSWORD
          valueFrom:
            secretKeyRef:
              name: hjtpx-secrets
              key: postgres-password
        - name: POSTGRES_DB
          value: hjtpx_db
        - name: PGDATA
          value: /var/lib/postgresql/data/pgdata
        ports:
        - containerPort: 5432
        volumeMounts:
        - name: postgres-storage
          mountPath: /var/lib/postgresql/data
        resources:
          requests:
            cpu: "500m"
            memory: "512Mi"
          limits:
            cpu: "1500m"
            memory: "1Gi"
        livenessProbe:
          exec:
            command:
            - pg_isready
            - -U
            - postgres
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          exec:
            command:
            - pg_isready
            - -U
            - postgres
          initialDelaySeconds: 5
          periodSeconds: 5
      volumes:
      - name: postgres-storage
        persistentVolumeClaim:
          claimName: postgres-pvc

---

apiVersion: v1
kind: Service
metadata:
  name: postgres
  namespace: hjtpx
spec:
  selector:
    app: postgres
  ports:
  - port: 5432
    targetPort: 5432
```

```bash
kubectl apply -f postgres.yaml
```

#### Q: 容器启动失败怎么办？

**症状表现**：
- `docker-compose ps` 显示容器状态为 `Exited`

- 健康检查失败

```bash
netstat -tlnp | grep 8080
lsof -i :8080

netstat -tlnp | grep 5432

netstat -tlnp | grep 6379
```

```bash

docker-compose config

# 检查环境变量
cat .env | grep -v "^#"
```

4. **检查依赖服务**
```bash
# 检查PostgreSQL
docker-compose ps postgres
docker-compose logs postgres | tail -20

# 检查Redis
docker-compose ps redis
docker-compose logs redis | tail -20
```

```bash
docker stats --no-stream
```

**常见问题与解决方案**：

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据库连接失败 | PostgreSQL未启动或配置错误 | 检查POSTGRES_HOST配置 |
| Redis连接失败 | Redis未启动或密码错误 | 检查REDIS_PASSWORD配置 |
| 端口被占用 | 8080端口已被其他服务占用 | 修改SERVER_PORT或停止冲突服务 |
| 内存不足 | 容器内存限制过低 | 增加容器内存或清理系统内存 |
| 磁盘空间不足 | 日志或数据占满磁盘 | 清理日志或增加磁盘空间 |
| 镜像拉取失败 | 网络问题或镜像不存在 | 检查网络或手动构建镜像 |
| 配置文件错误 | YAML格式错误 | 使用`docker-compose config`验证 |
| 权限不足 | 文件权限问题 | 检查挂载目录权限 |
| 环境变量缺失 | 必需的环境变量未设置 | 检查.env文件完整性 |

### 性能问题

#### Q: 响应速度慢怎么办？

**排查步骤**：

1. **检查系统资源**
```bash
# 查看资源使用
docker stats

# 查看详细CPU和内存
top
htop

# 检查磁盘IO
iostat -x 1 5
```

2. **检查数据库性能**
```bash
# 进入PostgreSQL
docker-compose exec postgres psql -U postgres -d hjtpx_db

# 查看慢查询
SELECT query, calls, mean_time, total_time
FROM pg_stat_statements
ORDER BY mean_time DESC
LIMIT 10;

# 查看连接数
SELECT count(*) FROM pg_stat_activity;
```

3. **检查Redis性能**
```bash
# 查看Redis状态
docker-compose exec redis redis-cli info

# 查看慢操作
docker-compose exec redis redis-cli slowlog get 10

# 查看大Key
docker-compose exec redis redis-cli --bigkeys
```

4. **检查应用日志**
```bash
# 查看慢请求日志
docker-compose logs app | grep -i "slow\|latency"

# 查看错误日志
docker-compose logs app | grep -i error
```

**优化建议**：

| 问题 | 解决方案 |
|------|----------|
| CPU使用率高 | 增加容器CPU限制，或水平扩展 |
| 内存不足 | 增加容器内存限制，或优化代码 |
| 数据库慢查询 | 添加索引，优化SQL |
| Redis响应慢 | 增加内存，使用集群模式 |
| 网络延迟 | 优化网络配置，使用CDN |
| 连接池满 | 增加连接池大小 |

### 数据问题

#### Q: 如何备份数据？

**备份PostgreSQL**：
```bash
# 创建备份
docker-compose exec postgres pg_dump -U postgres hjtpx_db > backup_$(date +%Y%m%d_%H%M%S).sql

# 压缩备份
docker-compose exec postgres pg_dump -U postgres hjtpx_db | gzip > backup_$(date +%Y%m%d).sql.gz
```

**备份Redis**：
```bash
# RDB快照备份
docker-compose exec redis redis-cli SAVE

# 复制RDB文件
docker cp $(docker-compose ps -q redis):/data/dump.rdb ./redis_backup.rdb

# AOF备份
docker-compose exec redis redis-cli BGREWRITEAOF
```

**自动化备份脚本**：
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/var/backups/hjtpx"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份PostgreSQL
docker-compose exec -T postgres pg_dump -U postgres hjtpx_db | gzip > $BACKUP_DIR/postgres_$DATE.sql.gz

# 备份Redis
docker cp $(docker-compose ps -q redis):/data/dump.rdb $BACKUP_DIR/redis_$DATE.rdb

# 保留最近30天备份
find $BACKUP_DIR -type f -mtime +30 -delete

echo "Backup completed: $DATE"
```

#### Q: 如何恢复数据？

**恢复PostgreSQL**：
```bash
# 恢复数据库
cat backup_20260518.sql | docker-compose exec -T postgres psql -U postgres -d hjtpx_db

# 或使用压缩文件
gunzip < backup_20260518.sql.gz | docker-compose exec -T postgres psql -U postgres -d hjtpx_db
```

**恢复Redis**：
```bash
# 停止Redis
docker-compose stop redis

# 复制备份文件
docker cp redis_backup.rdb $(docker-compose ps -q redis):/data/dump.rdb

# 启动Redis
docker-compose start redis
```

### 安全问题

#### Q: 如何防止DDoS攻击？

**配置Nginx限流**：
```nginx
# /etc/nginx/nginx.conf
http {
    limit_req_zone $binary_remote_addr zone=general:10m rate=10r/s;
    limit_req_zone $binary_remote_addr zone=login:10m rate=1r/s;
    
    server {
        location /api/v1/login {
            limit_req zone=login burst=5 nodelay;
        }
        
        location /api {
            limit_req zone=general burst=20 nodelay;
        }
    }
}
```

**配置应用层限流**：
```yaml
# backend/config/config.yaml
rate_limit:
  enabled: true
  default_limit: 100
  window_secs: 60
  
  rules:
    - path: "/api/v1/captcha/*"
      limit: 60
      window_secs: 60
    - path: "/api/v1/auth/login"
      limit: 10
      window_secs: 60
```

**配置云服务商DDoS防护**：
- AWS: 使用AWS Shield Standard
- 阿里云: 使用DDoS基础防护
- 腾讯云: 使用DDoS防护服务

### 升级问题

#### Q: 如何升级版本？

**Docker Compose升级**：
```bash
# 1. 备份数据
./scripts/backup.sh

# 2. 拉取最新代码
git pull

# 3. 停止服务
docker-compose down

# 4. 重新构建镜像
docker-compose build --no-cache

# 5. 启动服务
docker-compose up -d

# 6. 检查状态
docker-compose ps
docker-compose logs -f app
```

**Kubernetes升级**：
```bash
# 1. 更新镜像
kubectl set image deployment/hjtpx-app hjtpx-app=ghcr.io/opphk/hjtpx:new-version -n hjtpx

# 2. 查看滚动更新状态
kubectl rollout status deployment/hjtpx-app -n hjtpx

# 3. 验证功能
kubectl exec -it deployment/hjtpx-app -n hjtpx -- curl http://localhost:8080/health
```

**回滚**：
```bash
# Docker Compose
docker-compose down
git checkout v10.0
docker-compose up -d

# Kubernetes
kubectl rollout undo deployment/hjtpx-app -n hjtpx
```
