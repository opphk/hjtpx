## 简介

本指南提供 HJTPX 验证系统的安全加固最佳实践，涵盖系统安全、数据库安全、网络安全、应用安全等多个方面，帮助构建安全可靠的验证服务。

### 安全原则

1. **最小权限原则** - 只授予必要的最小权限
2. **纵深防御** - 多层防护措施
3. **默认拒绝** - 除明确允许的，否则都拒绝
4. **安全配置** - 禁用不安全的默认配置
5. **持续监控** - 建立完善的安全监控和告警

## 系统安全加固

### 操作系统加固

#### 1. 账户安全

```bash
# 禁用不必要的系统账户
usermod -L games
usermod -L news
usermod -L uucp
usermod -L proxy

# 限制 root 登录
echo "PermitRootLogin no" >> /etc/ssh/sshd_config

# 使用 sudo 替代 root
visudo
# 添加：your-user ALL=(ALL) ALL
```

#### 2. 密码策略

```bash
# 安装 libpam-pwquality
apt-get install libpam-pwquality

# 配置密码复杂度
vim /etc/security/pwquality.conf
# 添加或修改：
minlen = 12
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1

# 配置密码过期
vim /etc/login.defs
# 修改：
PASS_MAX_DAYS 90
PASS_MIN_DAYS 1
PASS_WARN_AGE 7
```

#### 3. SSH 加固

```bash
vim /etc/ssh/sshd_config

# 配置示例
Protocol 2
Port 2222  # 修改为非标准端口
LogLevel INFO
MaxAuthTries 3
ClientAliveInterval 300
ClientAliveCountMax 2
AllowUsers hjtpx-user
PasswordAuthentication no
PubkeyAuthentication yes
PermitEmptyPasswords no
X11Forwarding no
IgnoreRhosts yes
HostbasedAuthentication no
```

#### 4. 防火墙配置

```bash
# 安装 ufw
apt-get install ufw

# 配置防火墙规则
ufw default deny incoming
ufw default allow outgoing
ufw allow 2222/tcp   # SSH 端口
ufw allow 80/tcp     # HTTP
ufw allow 443/tcp    # HTTPS
ufw enable
```

### 文件系统加固

```bash
# 关键目录权限设置
chmod 700 /root
chmod 750 /opt/hjtpx
chmod 600 /etc/ssh/sshd_config

# 防止敏感文件被修改
chattr +i /etc/passwd
chattr +i /etc/shadow
chattr +i /etc/group

# 限制 /tmp 目录
mount -o remount,nosuid,nodev,noexec /tmp
# 添加到 /etc/fstab 永久生效
tmpfs /tmp tmpfs defaults,nosuid,nodev,noexec 0 0
```

### 安全更新

```bash
# 配置自动安全更新
apt-get install unattended-upgrades
dpkg-reconfigure -plow unattended-upgrades

# 手动更新检查
apt-get update
apt-get upgrade -s  # 模拟更新
```

## 数据库安全配置

### PostgreSQL 安全

#### 1. 访问控制

```bash
# 修改 pg_hba.conf
vim /var/lib/postgresql/16/main/pg_hba.conf

# 配置示例
local   all             postgres                                peer
local   all             all                                     peer
host    hjtpx_db       hjtpx_user      192.168.1.0/24          scram-sha-256
host    all             all             0.0.0.0/0               reject
```

#### 2. 用户权限

```sql
-- 创建专用用户
CREATE USER hjtpx_user WITH PASSWORD 'StrongPass123!';

-- 最小权限原则
CREATE DATABASE hjtpx_db OWNER hjtpx_user;
\c hjtpx_db

-- 只授予必要的权限
GRANT CONNECT ON DATABASE hjtpx_db TO hjtpx_user;
GRANT USAGE ON SCHEMA public TO hjtpx_user;
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO hjtpx_user;
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO hjtpx_user;

-- 撤销 public 权限
REVOKE ALL ON DATABASE hjtpx_db FROM public;
REVOKE ALL ON SCHEMA public FROM public;
```

#### 3. 启用 SSL

```bash
# 生成证书
openssl req -new -x509 -days 365 -nodes -text -out server.crt -keyout server.key -subj "/CN=your-domain.com"
chmod 600 server.key
chown postgres:postgres server.crt server.key

# 修改 postgresql.conf
vim /var/lib/postgresql/16/main/postgresql.conf
# 添加或修改：
ssl = on
ssl_cert_file = 'server.crt'
ssl_key_file = 'server.key'
password_encryption = scram-sha-256

# 重启 PostgreSQL
systemctl restart postgresql
```

#### 4. 审计日志

```sql
-- 启用日志收集
ALTER SYSTEM SET log_statement = 'ddl,mod';
ALTER SYSTEM SET log_connections = on;
ALTER SYSTEM SET log_disconnections = on;
ALTER SYSTEM SET log_duration = on;
ALTER SYSTEM SET log_line_prefix = '%t [%p]: [%c-%l] user=%u,db=%d,app=%a,client=%h ';

-- 重新加载配置
SELECT pg_reload_conf();
```

### 数据库加密

```sql
-- 敏感字段加密
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- 使用加密函数存储敏感数据
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username TEXT,
    password_hash TEXT  -- 存储哈希值，不存储明文
);

-- 插入时加密
INSERT INTO users (username, password_hash)
VALUES ('user1', crypt('StrongPass123!', gen_salt('bf')));

-- 验证密码
SELECT * FROM users WHERE username = 'user1' AND password_hash = crypt('StrongPass123!', password_hash);
```

## Redis 安全配置

### 基础安全配置

```bash
# 编辑 redis.conf
vim /etc/redis/redis.conf

# 关键配置
bind 127.0.0.1  # 只绑定本地地址
requirepass your-strong-redis-password  # 设置密码
protected-mode yes  # 启用保护模式
daemonize yes

# 禁用危险命令
rename-command FLUSHDB ""
rename-command FLUSHALL ""
rename-command CONFIG ""
rename-command SHUTDOWN ""
rename-command DEBUG ""
```

### Redis ACL（推荐）

```bash
# 使用 ACL 命令（Redis 6.0+）
redis-cli ACL SETUSER hjtpx_user on >StrongPass123! ~hjtpx:* +@all -@dangerous

# 保存 ACL 配置
redis-cli ACL SAVE
```

### Redis 传输加密

```bash
# 生成证书
openssl req -newkey rsa:4096 -x509 -days 365 -nodes -keyout redis.key -out redis.crt -subj "/CN=redis"

# 配置 redis.conf
tls-port 6379
port 0  # 禁用非加密端口
tls-cert-file /path/to/redis.crt
tls-key-file /path/to/redis.key
tls-ca-cert-file /path/to/ca.crt
tls-auth-clients yes
```

## 网络安全配置

### TLS/SSL 配置

#### 使用 Let's Encrypt 证书

```bash
# 安装 certbot
apt-get install certbot python3-certbot-nginx

# 获取证书
certbot certonly --standalone -d your-domain.com

# 自动续期
certbot renew --dry-run
```

#### Nginx TLS 配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # TLS 配置
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;
    ssl_session_tickets off;

    # HSTS
    add_header Strict-Transport-Security "max-age=63072000" always;

    # 安全头部
    add_header X-Frame-Options "DENY" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

  --from-literal=redis-password='strong-redis-pass' \

### JWT 安全配置

```yaml
# config.yaml
jwt:
  secret: "your-very-long-and-strong-secret-key-here-min-32-chars"
  expire_hours: 24
  algorithm: HS256  # 或 RS256 使用非对称加密

# 生产环境推荐使用 RSA 非对称加密
# 生成密钥对
openssl genrsa -out private.key 2048
openssl rsa -in private.key -pubout -out public.key
```

### 输入验证

应用中实现严格的输入验证：

```go
// 验证码验证示例
import (
    "github.com/go-playground/validator/v10"
    "regexp"
)

var validate *validator.Validate

func init() {
    validate = validator.New()
    validate.RegisterValidation("captcha_answer", validateCaptchaAnswer)
}

func validateCaptchaAnswer(fl validator.FieldLevel) bool {
    // 只允许数字字母，4-6位
    answer := fl.Field().String()
    match, _ := regexp.MatchString(`^[a-zA-Z0-9]{4,6}$`, answer)
    return match
}

type VerifyRequest struct {
    CaptchaID string `json:"captcha_id" validate:"required,uuid4"`
    Answer    string `json:"answer" validate:"required,captcha_answer"`
}
```

## 安全审计

### 日志审计

```yaml
# 审计日志配置
logging:
  level: info
  format: json
  include:
    - timestamp
    - level
    - request_id
    - user_id
    - ip_address
    - action
    - resource
    - status

  # 审计敏感操作
  audit_events:
    - login
    - logout
    - password_change
    - settings_change
    - privileged_action
```

### 监控告警

```yaml
# 安全告警规则
security_alerts:
  rules:
    # 暴力破解检测
    - name: brute_force
      condition: count(failed_login) > 10 within 5m
      severity: high
    
    # 异常请求检测
    - name: unusual_traffic
      condition: request_rate > (avg + 3*std_dev)
      severity: medium
    
    # 配置变更检测
    - name: config_change
      condition: config_file_modified
      severity: high
```

## 应急响应

### 事件响应流程

```
1. 发现和确认
   └─ 确认安全事件真实发生

2. 遏制和隔离
   └─ 隔离受影响的系统，防止事态扩大

3. 调查和分析
   └─ 收集证据，确定入侵来源和影响范围

4. 修复和恢复
   └─ 清除威胁，修复漏洞，恢复服务

5. 报告和总结
   └─ 记录事件，总结经验，改进防护措施
```
