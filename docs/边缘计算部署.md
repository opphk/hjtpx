# 边缘计算部署指南

## 概述

本文档描述了验证码系统边缘计算架构的部署和配置。边缘计算架构提供全球CDN边缘节点支持和边缘AI推理能力。

## 架构组件

### 1. 边缘节点管理 (Edge Node Manager)

边缘节点管理器负责管理全球分布的边缘计算节点。

#### 功能特性
- 节点注册与注销
- 心跳检测与健康监控
- 节点负载和延迟跟踪
- 基于地理位置的节点选择
- 集群统计和指标收集

#### 支持的区域
| 区域代码 | 位置 | CSP |
|---------|------|-----|
| ap-east-1 | 亚太东部 (香港) | AWS |
| eu-west-1 | 欧洲西部 (都柏林) | Azure |
| us-east-1 | 美洲东部 (弗吉尼亚) | GCP |
| cn-north-1 | 中国北部 (北京) | 阿里云 |

#### 使用示例

```go
// 创建边缘节点管理器
manager := edge.NewEdgeNodeManager(redisClient)

// 注册节点
node := &edge.EdgeNode{
    ID:        "node-001",
    Name:      "AP East Node 1",
    Region:    edge.RegionAP,
    IPAddress: "10.0.1.10",
    Port:      8080,
    Type:      edge.NodeTypeMixed,
    Capacity:  1000,
}
manager.RegisterNode(ctx, node)

// 选择最佳节点
selected, err := manager.SelectNode(&edge.NodeSelectorOptions{
    Region:   edge.RegionAP,
    NodeType: edge.NodeTypeMixed,
})
```

### 2. 智能DNS解析 (Edge DNS)

智能DNS系统提供基于地理位置的DNS解析和负载均衡。

#### 功能特性
- 地理DNS规则
- 健康检查与故障转移
- 多种负载均衡策略
- DNS缓存
- DNSSEC支持

#### 负载均衡策略
- **Round Robin**: 轮询分配
- **Weighted Round Robin**: 加权轮询
- **Latency Based**: 基于延迟
- **Geolocation Based**: 基于地理位置

#### 使用示例

```go
// 创建DNS解析器
resolver := edge.NewDNSResolver(zone, redisClient, nodeManager)

// 添加地理规则
rule := &edge.GeoDNSRule{
    Pattern:     "api.example.com",
    MatchType:   "region",
    Regions:     []edge.Region{edge.RegionAP},
    RecordValues: []string{"203.0.113.10"},
    Priority:    50,
    Enabled:     true,
}
resolver.AddRule(rule)

// 解析请求
response, err := resolver.Resolve(ctx, &edge.DNSQuery{
    Name:     "api.example.com",
    Type:     "A",
    ClientIP: "203.0.113.1",
})
```

### 3. 地理路由优化 (Geo Router)

地理路由系统提供智能流量路由和优化。

#### 功能特性
- 基于地理位置的路由
- 多策略路由选择
- 动态路由规则
- 流量监控和统计
- 规则优先级管理

#### 路由策略
- `latency`: 最低延迟
- `geographic`: 地理位置
- `load_balanced`: 负载均衡
- `failover`: 故障转移
- `performance`: 性能优先
- `cost_effective`: 成本优化

#### 使用示例

```go
// 创建地理路由器
router := edge.NewGeoRouter(nodeManager, redisClient)

// 添加路由规则
rule := &edge.RouteRule{
    Name:           "APAC Traffic",
    Priority:       50,
    Strategy:       edge.RoutingGeo,
    SourceRegions:  []edge.Region{edge.RegionAP, edge.RegionJP},
    TargetRegions:  []edge.Region{edge.RegionAP},
    Weight:         100,
    Enabled:        true,
}
router.AddRule(rule)

// 路由请求
response, err := router.Route(ctx, &edge.RouteRequest{
    SourceIP:     "203.0.113.1",
    TargetType:   edge.NodeTypeMixed,
    Strategy:     edge.RoutingGeo,
    IncludeBackup: true,
})
```

### 4. 边缘健康检查 (Health Check)

健康检查系统监控边缘节点和服务状态。

#### 功能特性
- TCP/HTTP/HTTPS健康检查
- DNS解析检查
- 连续失败检测
- 节点状态管理
- 历史记录和统计

#### 检查类型
- `tcp`: TCP连接检查
- `http`: HTTP请求检查
- `https`: HTTPS请求检查
- `dns`: DNS解析检查
- `icmp`: ICMP ping检查

#### 使用示例

```go
// 创建健康检查管理器
healthManager := edge.NewHealthCheckManager(nodeManager, redisClient)

// 添加健康检查
check := &edge.HealthCheck{
    NodeID:   "node-001",
    Target:   "10.0.1.10",
    Port:     8080,
    Type:     edge.CheckTypeHTTP,
    Interval: 30 * time.Second,
    Timeout:  5 * time.Second,
    Threshold: 3,
}
healthManager.AddCheck(check)

// 获取节点健康状态
health, err := healthManager.GetNodeHealth("node-001")
```

### 5. 边缘AI推理引擎 (Edge AI Engine)

边缘AI推理引擎提供本地AI模型推理能力。

#### 功能特性
- 模型注册和管理
- 模型加载和卸载
- 异步推理
- 批量推理
- 推理结果缓存
- 模型指标收集

#### 支持的任务类型
- `classification`: 分类
- `detection`: 目标检测
- `segmentation`: 图像分割
- `nlp`: 自然语言处理
- `regression`: 回归
- `anomaly`: 异常检测

#### 使用示例

```go
// 创建AI引擎
aiEngine := edge.NewEdgeAIEngine(nodeManager, redisClient, compressor)

// 注册模型
model := &edge.ModelInfo{
    ID:         "fraud-detect-v1",
    Name:       "Fraud Detection Model",
    Version:    "1.0.0",
    Type:       edge.ModelTypeONNX,
    TaskType:   edge.TaskTypeAnomaly,
    InputShape: []int{1, 100},
    OutputShape: []int{1},
}
aiEngine.RegisterModel(ctx, model)

// 执行推理
response, err := aiEngine.Infer(ctx, &edge.InferenceRequest{
    ModelID:   "fraud-detect-v1",
    InputData: inputData,
    Options: &edge.InferenceOptions{
        BatchSize:      32,
        UseCache:       true,
        ConfidenceThreshold: 0.8,
    },
})
```

### 6. 模型压缩优化 (Model Compressor)

模型压缩工具优化AI模型以适应边缘设备。

#### 压缩方法
- **Pruning**: 权重剪枝
- **Quantization**: 量化 (FP16, INT8, INT4)
- **Distillation**: 知识蒸馏
- **Factorization**: 矩阵分解

#### 使用示例

```go
// 创建模型压缩器
compressor := edge.NewModelCompressor(redisClient)

// 量化压缩
config := &edge.CompressionConfig{
    Method:          edge.MethodQuantization,
    QuantizationType: edge.QuantINT8,
    PreserveAccuracy: true,
}
result, err := compressor.Compress(ctx, modelID, modelData, config)

// 自动压缩到目标大小
autoResult, err := compressor.AutoCompress(ctx, modelID, modelData, 50.0) // 50MB
```

### 7. 边缘缓存同步 (Edge Cache Sync)

缓存同步系统确保边缘节点间的数据一致性。

#### 同步策略
- `eager`: 即时同步
- `lazy`: 懒同步
- `write_through`: 写穿透
- `write_behind`: 写回
- `eventual`: 最终一致

#### 使用示例

```go
// 创建缓存同步器
cacheSync := edge.NewEdgeCacheSync(nodeManager, redisClient, &edge.CacheSyncConfig{
    Strategy:          edge.SyncStrategyEventual,
    BatchSize:         100,
    SyncInterval:      5 * time.Second,
    CompressionEnabled: true,
})

// 设置缓存
cacheSync.Set(ctx, "key1", value, 5*time.Minute, edge.WithTags([]string{"user", "profile"}))

// 获取缓存
value, err := cacheSync.Get(ctx, "key1")

// 按标签失效
cacheSync.InvalidateByTag(ctx, "user")
```

### 8. 边缘负载均衡 (Edge Load Balancer)

负载均衡器分配流量到最佳边缘节点。

#### 负载均衡算法
- `round_robin`: 轮询
- `least_connection`: 最小连接
- `ip_hash`: IP哈希
- `weighted`: 加权
- `latency_based`: 基于延迟
- `resource_based`: 基于资源
- `geolocation`: 基于地理位置
- `adaptive`: 自适应

#### 使用示例

```go
// 创建负载均衡器
lb := edge.NewEdgeLoadBalancer(nodeManager, healthCheck, redisClient)

// 添加节点
lbNode := &edge.LBNode{
    NodeID:    "node-001",
    IPAddress: "10.0.1.10",
    Port:      8080,
    Weight:    100,
}
lb.AddNode(lbNode)

// 转发请求
response, err := lb.ForwardRequest(ctx, &edge.LBRequest{
    ClientIP: "203.0.113.1",
    Path:     "/api/verify",
    Method:   "POST",
    Policy:   "adaptive",
})
```

## 部署

### 前置条件
- Go 1.21+
- Redis 6.0+
- PostgreSQL 13+ (可选)
- 支持的操作系统: Linux, macOS, Windows

### 构建

```bash
cd /workspace/hjtpx/backend
go build -o bin/hjtpx ./cmd/server
```

### 运行

```bash
./bin/hjtpx --config config/config.yaml
```

### Docker部署

```dockerfile
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o hjtpx ./cmd/server

FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /app/hjtpx .
COPY --from=builder /app/config ./config
EXPOSE 8080
CMD ["./hjtpx"]
```

## 故障排除

### 节点不可用
1. 检查网络连接
2. 验证健康检查配置
3. 查看节点日志
4. 检查资源使用情况

### DNS解析失败
1. 验证DNS服务器配置
2. 检查区域配置
3. 确认记录健康状态
4. 查看DNS缓存

### AI推理超时
1. 检查模型大小和压缩配置
2. 验证网络延迟
3. 调整超时配置
4. 检查推理引擎状态

## API端点

### 节点管理
- `GET /api/v1/edge/nodes` - 列出所有节点
- `GET /api/v1/edge/nodes/:id` - 获取节点详情
- `POST /api/v1/edge/nodes` - 注册节点
- `DELETE /api/v1/edge/nodes/:id` - 注销节点

### DNS
- `GET /api/v1/edge/dns/resolve` - DNS解析
- `GET /api/v1/edge/dns/zones` - 列出DNS区域
- `POST /api/v1/edge/dns/rules` - 添加规则

### 路由
- `GET /api/v1/edge/routes` - 获取路由规则
- `POST /api/v1/edge/routes` - 添加路由规则
- `POST /api/v1/edge/routes/calculate` - 计算路由

### 健康检查
- `GET /api/v1/edge/health` - 健康状态
- `GET /api/v1/edge/health/:node_id` - 节点健康详情
- `POST /api/v1/edge/health/:check_id/trigger` - 触发检查

### AI推理
- `POST /api/v1/edge/ai/infer` - 执行推理
- `GET /api/v1/edge/ai/models` - 列出模型
- `POST /api/v1/edge/ai/models` - 注册模型
- `POST /api/v1/edge/ai/compress` - 压缩模型

### 缓存
- `GET /api/v1/edge/cache/:key` - 获取缓存
- `POST /api/v1/edge/cache/:key` - 设置缓存
- `DELETE /api/v1/edge/cache/:key` - 删除缓存

### 负载均衡
- `GET /api/v1/edge/lb/nodes` - 列出LB节点
- `POST /api/v1/edge/lb/forward` - 转发请求
- `GET /api/v1/edge/lb/policies` - 列出策略
