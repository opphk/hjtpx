## 快速开始

### 环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Docker | 20.10+ | 24.0+ |
| Docker Compose | 2.0+ | 2.20+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 20GB | 50GB+ |

### 一键部署

```bash
# 克隆项目
git clone https://github.com/opphk/hjtpx.git
cd hjtpx

# 配置环境变量
cp .env.example .env
vim .env  # 修改密码等敏感配置

# 执行部署
chmod +x scripts/*.sh
./scripts/deploy.sh
```

### 部署验证

```bash
# 检查服务状态
docker-compose ps

# 运行健康检查
./scripts/health-check.sh

# 访问服务
curl http://localhost:8080/health
```

## 生产环境部署

### 部署流程

```
1. 环境准备
   ├── 系统要求检查
   ├── Docker 安装
   └── 依赖服务安装

2. 配置管理
   ├── 环境变量配置
   ├── SSL 证书配置
   └── 备份策略配置

3. 部署执行
   ├── 备份当前版本
   ├── 构建镜像
   ├── 启动服务
   └── 健康检查

4. 验证测试
   ├── API 功能测试
   ├── 性能基准测试
   └── 安全扫描
```

### 系统要求

| 部署规模 | CPU | 内存 | 磁盘 | QPS |
|---------|-----|------|------|-----|
| 开发/测试 | 2核 | 4GB | 20GB | <100 |
| 小型生产 | 4核 | 8GB | 50GB | 100-500 |
| 中型生产 | 8核 | 16GB | 100GB | 500-2000 |
| 大型生产 | 16核+ | 32GB+ | 500GB+ | >2000 |

### 生产部署脚本

```bash
#!/bin/bash
# 生产环境部署

set -e

# 1. 备份
./scripts/rollback.sh create

# 2. 停止服务
docker-compose down

# 3. 构建镜像
docker-compose build --no-cache

# 4. 启动服务
docker-compose up -d

# 5. 等待就绪
sleep 30

# 6. 健康检查
./scripts/health-check.sh

# 7. 验证
curl -f http://localhost:8080/health
```

### Nginx 配置

```nginx
upstream hjtpx_backend {
    server 127.0.0.1:8080;
    keepalive 32;
}

server {
    listen 80;
    server_name your-domain.com;

    # 强制 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/ssl/certs/your-cert.pem;
    ssl_certificate_key /etc/ssl/private/your-key.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;

    # 限流配置
    limit_req_zone $binary_remote_addr zone=api:10m rate=100r/s;

    location / {
        proxy_pass http://hjtpx_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_connect_timeout 30s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;

        limit_req zone=api burst=50 nodelay;
    }

    location /health {
        proxy_pass http://hjtpx_backend;
        access_log off;
    }
}
```

### systemd 服务配置

```ini
[Unit]

```

## Kubernetes 部署

### Helm Chart 结构

```yaml
# values.yaml
replicaCount: 3

image:
  repository: ghcr.io/opphk/hjtpx
  tag: latest
  pullPolicy: IfNotPresent

service:
  type: ClusterIP
  port: 80
  targetPort: 8080

resources:
  limits:
    cpu: 1000m
    memory: 512Mi
  requests:
    cpu: 500m
    memory: 256Mi

autoscaling:
  enabled: true
  minReplicas: 3
  maxReplicas: 10
  targetCPUUtilizationPercentage: 70

ingress:
  enabled: true
  className: nginx
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
  hosts:
    - host: your-domain.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - hosts:
        - your-domain.com
      secretName: hjtpx-tls

postgresql:
  enabled: true
  auth:
    database: hjtpx_db
    username: postgres
    password: your-password

redis:
  enabled: true
  auth:
    password: your-password
```

### 部署命令

```bash
# 添加 Helm 仓库
helm repo add hjtpx https://charts.hjtpx.io
helm repo update

# 安装
helm install hjtpx hjtpx/hjtpx -n hjtpx --create-namespace \
  --set postgresql.auth.password=your-password \
  --set redis.auth.password=your-password \
  --values values.yaml

# 升级
helm upgrade hjtpx hjtpx/hjtpx -n hjtpx --values values.yaml

# 回滚
helm rollback hjtpx -n hjtpx
```

## 环境配置

### 必需环境变量

| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|-----|
| `POSTGRES_PASSWORD` | PostgreSQL 密码 | - | 是 |
| `POSTGRES_DB` | 数据库名 | hjtpx_db | 否 |
| `REDIS_PASSWORD` | Redis 密码 | - | 否 |
| `JWT_SECRET` | JWT 密钥 (≥32字符) | - | 是 |

### 可选环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `GIN_MODE` | 运行模式 | release |
| `SERVER_PORT` | 服务端口 | 8080 |
| `LOG_LEVEL` | 日志级别 | info |
| `MAX_OPEN_CONNS` | 最大连接数 | 100 |
| `CACHE_TTL` | 缓存 TTL (秒) | 3600 |

## 回滚操作

### 创建备份

```bash
# 创建当前版本备份
./scripts/rollback.sh create

# 查看备份列表
./scripts/rollback.sh list

# 查看备份状态
./scripts/rollback.sh status
```

### 执行回滚

```bash
# 快速回滚到上一版本
./scripts/rollback.sh quick

# 回滚到指定备份
./scripts/rollback.sh restore backups/rollback_20260518_120000.tar.gz
```

### 回滚机制说明

```
备份包含:
├── Docker 镜像
├── docker-compose.yml
├── .env 配置
├── PostgreSQL 数据
└── Redis 数据

保留策略:
├── 默认保留 5 个版本
└── 可通过 KEEP_VERSIONS 环境变量调整
```

## 故障排查

### 常见问题

#### 1. 服务启动失败

```bash
# 查看详细日志
docker-compose logs -f app

# 检查端口占用
netstat -tlnp | grep 8080

# 检查依赖服务
docker-compose ps postgres redis
```

#### 2. 数据库连接失败

```bash
# 检查 PostgreSQL 状态
docker-compose exec postgres pg_isready -U postgres

# 测试连接
docker-compose exec app sh -c 'PGPASSWORD=$POSTGRES_PASSWORD psql -h postgres -U postgres -d hjtpx_db -c "SELECT 1"'

# 查看连接数
docker-compose exec postgres psql -U postgres -d hjtpx_db -c "SELECT count(*) FROM pg_stat_activity"
```

#### 3. Redis 连接失败

```bash
# 检查 Redis 状态
docker-compose exec redis redis-cli ping

# 查看 Redis 日志
docker-compose logs redis

# 清除缓存测试
docker-compose exec redis redis-cli FLUSHALL
```

#### 4. 性能问题

```bash
# 查看资源使用
docker stats

# 检查慢查询
docker-compose exec postgres psql -U postgres -d hjtpx_db -c \
  "SELECT query, calls, mean_time FROM pg_stat_statements ORDER BY mean_time DESC LIMIT 10"

# 查看 Redis 大 Key
docker-compose exec redis redis-cli --bigkeys
```

### 健康检查

```bash
# 运行完整健康检查
./scripts/health-check.sh

# 检查项:
# ✓ 后端服务响应
# ✓ 容器运行状态
# ✓ 数据库连接
# ✓ Redis 连接
# ✓ API 端点可用性
# ✓ 系统资源使用
```

### 日志分析

```bash
# 实时日志
docker-compose logs -f --tail=100

# 错误日志
docker-compose logs app 2>&1 | grep -i error

# 访问日志统计
docker-compose logs nginx 2>&1 | awk '{print $9}' | sort | uniq -c | sort -rn | head -20
```

```bash

    daily
    rotate 30

    delaycompress
    notifempty
```
