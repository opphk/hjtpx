## 日常运维

### 服务状态检查

```bash
# 检查所有服务状态 (Docker Compose)
docker-compose ps

# 检查健康状态
curl http://localhost:8080/health

# 检查容器资源使用
docker stats

# 查看服务启动时间
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Created}}"
```

### 日志管理

```bash
# 查看实时日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f nginx

# 日志轮转配置
logrotate -f /etc/logrotate.d/hjtpx
```

### 性能监控

```bash
# 查看资源使用
docker stats --no-stream

# Prometheus 指标
curl http://localhost:9090/metrics

# 查看应用日志中的慢请求
docker-compose logs backend | grep -i "slow\|timeout"
```

## 故障处理

### 服务不可用

**排查步骤：**

1. **检查服务状态**
```bash
docker-compose ps
docker-compose logs backend
```

2. **检查端口占用**
```bash
netstat -tlnp | grep 8080
lsof -i :8080
```

3. **检查依赖服务**
```bash
docker-compose exec postgres pg_isready -U postgres
docker-compose exec redis redis-cli ping
```

4. **重启服务**
```bash
docker-compose restart backend
```

### 数据库连接失败

**排查步骤：**

1. **检查 PostgreSQL 状态**
```bash
docker-compose exec postgres pg_isready -U postgres
docker-compose logs postgres | tail -20
```

2. **检查网络连接**
```bash
docker-compose exec backend ping postgres
docker-compose exec backend nc -zv postgres 5432
```

3. **检查权限配置**
```bash
docker-compose exec postgres psql -U postgres -c "\l"
```

4. **检查连接数**
```bash
docker-compose exec postgres psql -U postgres -c "SELECT count(*) FROM pg_stat_activity;"
```

### 缓存失效

**排查步骤：**

1. **检查 Redis 状态**
```bash
docker-compose exec redis redis-cli ping
docker-compose exec redis redis-cli info
```

2. **检查内存使用**
```bash
docker-compose exec redis redis-cli info memory
docker stats redis
```

3. **清理过期缓存**
```bash
docker-compose exec redis redis-cli FLUSHDB
```

4. **检查 Redis 日志**
```bash
docker-compose logs redis | tail -50
```

### 性能下降

**排查步骤：**

1. **查看系统资源**
```bash
top
free -h
df -h
```

2. **查看慢查询**
```bash
docker-compose exec postgres psql -U postgres -d hjtpx_db -c "
SELECT query, calls, mean_time, total_time
FROM pg_stat_statements
ORDER BY mean_time DESC
LIMIT 10;
"
```

3. **查看应用指标**
```bash
curl http://localhost:8080/metrics
```

4. **重启服务释放资源**
```bash
docker-compose restart backend
```

## 升级维护

### 版本升级

**标准升级流程：**

1. **备份数据**
```bash
./scripts/backup.sh
```

2. **拉取最新代码**
```bash
git pull origin main
```

3. **重新构建**
```bash
docker-compose build --no-cache
```

4. **启动服务**
```bash
docker-compose up -d
```

5. **验证服务**
```bash
curl http://localhost:8080/health
```

6. **检查日志**
```bash
docker-compose logs -f backend
```

### 数据库迁移

**迁移步骤：**

1. **查看迁移脚本**
```bash
ls -la scripts/migrate/
```

2. **执行迁移**
```bash
docker-compose exec backend go run ./scripts/migrate.go
```

3. **验证迁移**
```bash
docker-compose exec postgres psql -U postgres -d hjtpx_db -c "\dt"
```

4. **回滚（如需要）**
```bash
docker-compose exec backend go run ./scripts/migrate.go --rollback
```

### 配置更新

**热更新配置：**

1. **修改配置文件**
```bash
vim backend/config/config.yaml
```

2. **发送重载信号**
```bash
docker-compose exec backend kill -HUP 1
```

3. **验证配置**
```bash
curl http://localhost:8080/health
```

## 备份恢复

### 自动备份

**配置 crontab：**
```bash
# 每天凌晨2点执行备份
0 2 * * * /opt/hjtpx/scripts/backup.sh >> /var/log/hjtpx-backup.log 2>&1

# 每周日凌晨3点执行全量备份
0 3 * * 0 /opt/hjtpx/scripts/backup.sh full >> /var/log/hjtpx-backup.log 2>&1
```

### 数据库备份

```bash
# 基本备份
pg_dump -h localhost -U postgres hjtpx_db > backup.sql

# 压缩备份
pg_dump -h localhost -U postgres hjtpx_db | gzip > backup_$(date +%Y%m%d).sql.gz

# 备份到远程服务器
pg_dump -h localhost -U postgres hjtpx_db | ssh user@backup-server "cat > /backup/hjtpx_$(date +%Y%m%d).sql"
```

### Redis 备份

```bash
# 保存当前数据
docker-compose exec redis redis-cli SAVE

# 复制备份文件
docker-compose cp redis:/data/dump.rdb ./redis_backup_$(date +%Y%m%d).rdb

# 压缩备份
gzip redis_backup_$(date +%Y%m%d).rdb
```

### 数据恢复

**恢复数据库：**
```bash
# 解压备份（如需要）
gunzip backup_20240516.sql.gz

# 恢复数据
psql -h localhost -U postgres -d hjtpx_db < backup_20240516.sql

# Docker 环境恢复
gunzip < backup_20240516.sql.gz | docker-compose exec -T postgres psql -U postgres hjtpx_db
```

**恢复 Redis：**
```bash
# 复制备份文件
docker-compose cp redis_backup_20240516.rdb redis:/data/dump.rdb

# 重启 Redis
docker-compose restart redis

# 或使用 BGSAVE
docker-compose exec redis redis-cli BGSAVE
```

### 定时清理

```bash
# 清理30天前的备份
find /opt/hjtpx/backups -mtime +30 -delete

# 清理日志文件
find /var/log/hjtpx -mtime +30 -delete
```

## 性能调优

### PostgreSQL 优化

**连接池配置：**
```yaml
# backend/config/config.yaml
database:
  max_open_conns: 100
  max_idle_conns: 10
  conn_max_lifetime: 3600
```

**SQL 优化：**
```sql
-- 创建索引
CREATE INDEX idx_verification_session ON verification_log(session_id);
CREATE INDEX idx_verification_created ON verification_log(created_at);

-- 分析慢查询
SELECT query, calls, mean_time, total_time
FROM pg_stat_statements
ORDER BY mean_time DESC
LIMIT 10;

-- 查看连接数
SELECT count(*) FROM pg_stat_activity;
```

**系统参数优化：**
```sql
ALTER SYSTEM SET shared_buffers = '256MB';
ALTER SYSTEM SET effective_cache_size = '1GB';
ALTER SYSTEM SET work_mem = '64MB';
ALTER SYSTEM SET maintenance_work_mem = '256MB';
SELECT pg_reload_conf();
```

### Redis 优化

**配置文件：**
```conf
maxmemory 256mb
maxmemory-policy allkeys-lru
appendonly yes
appendfsync everysec
save 900 1
save 300 10
save 60 10000
```

**监控命令：**
```bash
redis-cli INFO stats
redis-cli INFO memory
redis-cli INFO clients
redis-cli INFO persistence
```

### Nginx 优化

**连接优化：**
```nginx
worker_processes auto;
worker_connections 1024;
use epoll;
multi_accept on;

keepalive_timeout 65;
keepalive_requests 10000;

gzip on;
gzip_types text/plain text/css application/json;
```

**缓存优化：**
```nginx
open_file_cache max=10000 inactive=30s;
open_file_cache_valid 60s;
open_file_cache_min_uses 2;
```

### Go 应用优化

**Goroutine 控制：**
```go
// 限制并发数
var sem = make(chan struct{}, 100)

func handler() {
    sem <- struct{}{}
    defer func() { <-sem }()
    // 处理请求
}
```

**内存优化：**
```go
// 使用 sync.Pool 复用对象
var bufferPool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 4096)
    },
}
```

## 日志管理

### 日志配置

**日志级别：**
| 级别 | 说明 | 场景 |
|------|------|------|
| DEBUG | 调试信息 | 开发环境 |
| INFO | 一般信息 | 正常操作 |
| WARN | 警告信息 | 需要关注 |
| ERROR | 错误信息 | 需要处理 |

**日志格式：**
```json
{
  "time": "2024-05-16T12:00:00Z",
  "level": "INFO",
  "service": "hjtpx",
  "message": "request processed",
  "request_id": "uuid",
  "method": "POST",
  "path": "/api/v1/captcha/verify",
  "status": 200,
  "latency": "45ms",
  "ip": "192.168.1.1"
}
```

### 日志查看

**应用日志：**
```bash
# 查看所有日志
docker-compose logs backend

# 实时查看
docker-compose logs -f backend

# 查看最近100行
docker-compose logs --tail 100 backend

# 搜索错误
docker-compose logs backend | grep -i error

# 搜索特定请求
docker-compose logs backend | grep "request_id"
```

**Nginx 日志：**
```bash
# 访问日志
tail -f /var/log/nginx/access.log

# 错误日志
tail -f /var/log/nginx/error.log

# 统计状态码
awk '{print $9}' /var/log/nginx/access.log | sort | uniq -c
```

**PostgreSQL 日志：**
```bash
docker-compose logs postgres | grep -i "error\|warning"
```

### 日志分析

**常用命令：**
```bash
# 统计请求量
wc -l access.log

# 统计 IP 访问量
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -20

# 统计接口访问量
awk -F'"' '{print $2}' access.log | awk '{print $2}' | sort | uniq -c

# 统计响应时间
awk -F'"' '{print $NF}' access.log | awk '{print $2}' | sort -n

# 统计错误请求
grep " 5[0-9][0-9] " access.log | wc -l
```

### 日志轮转

**logrotate 配置：**
```bash
# /etc/logrotate.d/hjtpx
/var/log/hjtpx/*.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    create 0640 root root
    sharedscripts
    postrotate
        docker-compose -f /opt/hjtpx/docker-compose.yml restart backend
    endscript
}
```
