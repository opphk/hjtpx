## 配置方式

系统支持多种配置方式，优先级从高到低：

1. **环境变量**（最高优先级）
2. **配置文件**（YAML 格式）
3. **默认值**（最低优先级）

### 配置文件位置

- 默认位置：`backend/config/config.yaml`
- 可通过环境变量 `CONFIG_PATH` 指定自定义位置

## 配置项详解

### 服务器配置 (server)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| port | int | 8080 | 服务监听端口 |
| mode | string | debug | 运行模式，可选值：debug、release、test |
| read_timeout | int | 30 | 读取超时时间（秒） |
| write_timeout | int | 30 | 写入超时时间（秒） |
| idle_timeout | int | 60 | 空闲超时时间（秒） |
| max_body_size | int | 10 | 最大请求体大小（MB） |

**说明**：
- 生产环境必须使用 `release` 模式
- 根据实际网络情况调整超时时间

---

### PostgreSQL 配置 (postgres)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| host | string | localhost | 数据库主机地址 |
| port | int | 5432 | 数据库端口 |
| user | string | postgres | 数据库用户名 |
| password | string | - | 数据库密码 |
| db_name | string | verification | 数据库名称 |
| sslmode | string | disable | SSL 模式 |
| max_open_conns | int | 100 | 最大打开连接数 |
| max_idle_conns | int | 20 | 最大空闲连接数 |
| conn_max_lifetime | int | 300 | 连接最大生命周期（秒） |
| conn_max_idle_time | int | 60 | 连接最大空闲时间（秒） |

**sslmode 说明**：
- `disable`：不使用 SSL（开发环境）
- `require`：使用 SSL，但不验证证书（生产环境慎用）
- `verify-ca`：使用 SSL，验证 CA 证书
- `verify-full`：使用 SSL，完整验证（推荐生产环境）

**连接池配置建议**：
- `max_open_conns`：通常设置为 CPU 核心数的 10-20 倍
- `max_idle_conns`：通常设置为 `max_open_conns` 的 20%-30%
- `conn_max_lifetime`：建议 300-1800 秒

### Redis 配置 (redis)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| host | string | localhost | Redis 主机地址 |
| port | int | 6379 | Redis 端口 |
| password | string | - | Redis 密码 |
| db | int | 0 | Redis 数据库编号 |
| max_retries | int | 3 | 最大重试次数 |
| dial_timeout | int | 5 | 拨号超时时间（秒） |
| read_timeout | int | 3 | 读取超时时间（秒） |
| write_timeout | int | 3 | 写入超时时间（秒） |
| pool_size | int | 100 | 连接池大小 |
| min_idle_conns | int | 10 | 最小空闲连接数 |
| pool_timeout | int | 30 | 连接池超时时间（秒） |

**哨兵模式配置**：
```yaml
redis:
  sentinel:
    enabled: true
    master_name: "mymaster"
    addresses:
      - "192.168.1.10:26379"
      - "192.168.1.11:26379"
    password: "your-sentinel-password"
```

**集群模式配置**：
```yaml
redis:
  cluster:
    enabled: true
    addresses:
      - "192.168.1.10:6379"
      - "192.168.1.11:6379"
      - "192.168.1.12:6379"
    password: "your-redis-password"
```

### JWT 配置 (jwt)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| secret | string | - | JWT 密钥（必填） |
| expire_hours | int | 24 | Token 过期时间（小时） |
| refresh_hours | int | 168 | Token 刷新时间（小时） |
| issuer | string | hjtpx-verification | 签发者 |
| audience | string | hjtpx-users | 受众 |

**安全建议**：
- `secret` 长度至少 32 个字符
- 使用强随机字符串
- 定期更换密钥
- 不要将密钥提交到代码仓库

**生成安全密钥的方法**：
```bash
# 使用 OpenSSL
openssl rand -hex 32

# 或使用 Python
python -c "import secrets; print(secrets.token_hex(32))"
```

### 安全配置 (security)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| enable_csrf | bool | true | 是否启用 CSRF 保护 |
| csrf_expire_secs | int | 3600 | CSRF Token 过期时间（秒） |
| enable_xss | bool | true | 是否启用 XSS 保护 |
| enable_signature | bool | true | 是否启用请求签名验证 |
| signature_expire_secs | int | 300 | 签名过期时间（秒） |
| signature_key | string | - | 签名密钥 |
| whitelist_ips | []string | [] | IP 白名单 |
| blacklist_ips | []string | [] | IP 黑名单 |
| enable_rate_limit | bool | true | 是否启用速率限制 |

**CORS 配置**：
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| enabled | bool | true | 是否启用 CORS |
| allow_origins | []string | ["*"] | 允许的源 |
| allow_methods | []string | ["GET","POST","PUT","DELETE","OPTIONS"] | 允许的方法 |
| allow_headers | []string | ["*"] | 允许的头部 |
| expose_headers | []string | [] | 暴露的头部 |
| allow_credentials | bool | false | 是否允许凭证 |
| max_age | int | 86400 | 预检请求缓存时间（秒） |

**生产环境 CORS 建议**：
```yaml
cors:
  enabled: true
  allow_origins:
    - "https://your-domain.com"
    - "https://www.your-domain.com"
  allow_methods: ["GET", "POST", "OPTIONS"]
  allow_headers: ["Content-Type", "Authorization", "X-Request-ID"]
  expose_headers: ["X-Request-ID"]
  allow_credentials: true
  max_age: 86400
```

## 环境变量映射

所有配置项都可以通过环境变量覆盖，命名规则：

- 使用大写字母
- 使用下划线分隔单词
- 配置层级使用双下划线分隔

**示例**：

| 配置文件路径 | 环境变量 |
|--------------|----------|
| server.port | SERVER_PORT |
| postgres.host | POSTGRES_HOST |
| postgres.password | POSTGRES_PASSWORD |
| redis.host | REDIS_HOST |
| redis.password | REDIS_PASSWORD |
| jwt.secret | JWT_SECRET |
| jwt.expire_hours | JWT_EXPIRE_HOURS |
| security.enable_csrf | SECURITY_ENABLE_CSRF |
| logging.level | LOGGING_LEVEL |
| rate_limit.default_limit | RATE_LIMIT_DEFAULT_LIMIT |

**完整环境变量列表**：

```bash
# 服务器
SERVER_PORT=8080
SERVER_MODE=release

# PostgreSQL
POSTGRES_HOST=localhost

POSTGRES_USER=postgres
POSTGRES_PASSWORD=your-password

# 速率限制
```

## 安全配置建议

### 1. 密钥管理

**不要将密钥提交到代码仓库**：
- 使用环境变量
- 使用密钥管理服务（如 AWS KMS、HashiCorp Vault）
- 使用 Kubernetes Secrets

**示例：使用 Kubernetes Secrets**：
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: hjtpx-secrets
type: Opaque
stringData:
  postgres-password: "your-strong-password"
  redis-password: "your-strong-redis-password"
  jwt-secret: "your-very-secure-jwt-secret"
```

### 2. 数据库安全

- 使用强密码
- 启用 SSL/TLS
- 限制数据库用户权限
- 使用单独的数据库用户
- 定期轮换密码

### 3. Redis 安全

- 设置密码
- 绑定到特定网络接口
- 禁用危险命令（如 FLUSHDB、FLUSHALL）
- 使用 SSL/TLS（Redis 6+）

**Redis 配置示例**：
```conf
requirepass your-strong-redis-password
bind 127.0.0.1 192.168.1.100
protected-mode yes
rename-command FLUSHDB ""
rename-command FLUSHALL ""
rename-command CONFIG ""
```

### 4. 网络安全

- 使用防火墙限制访问
- 启用 HTTPS
- 使用安全协议（TLS 1.2+）
- 配置安全头部

**Nginx 安全头部配置**：
```nginx
add_header X-Frame-Options "DENY" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Content-Security-Policy "default-src 'self'" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
```

## 常见问题

### Q: 如何验证配置是否正确？

A: 使用健康检查端点：
```bash
curl http://localhost:8080/health
```

### Q: 如何热更新配置？

A: 目前需要重启服务。支持热更新的配置会在未来版本中提供。

### Q: 配置文件中的密码可以加密吗？

A: 可以。建议使用环境变量或密钥管理服务，不要在配置文件中存储明文密码。

### Q: 如何在多环境中管理配置？

A: 使用多个配置文件（config.dev.yaml, config.prod.yaml）或使用环境变量。
