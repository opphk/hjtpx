## 快速开始

### 方式一：Docker Compose 快速部署

```bash
# 1. 克隆项目
git clone https://github.com/opphk/hjtpx.git
cd hjtpx

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件设置密码

# 3. 启动服务
docker-compose up -d

# 4. 访问服务
# 用户端: http://localhost
# 管理后台: http://localhost/admin
# 默认账号: admin / admin123
```

### 方式二：手动部署

```bash
# 1. 安装依赖
# Go 1.25+, PostgreSQL 12+, Redis 6+

# 2. 克隆项目
git clone https://github.com/opphk/hjtpx.git
cd hjtpx/backend

# 3. 安装依赖
go mod download

# 4. 配置数据库
# 编辑 config/config.yaml 或设置环境变量

# 5. 启动服务
go run ./cmd/api/main.go

# 6. 访问 http://localhost:8080
```

## 用户端集成

### 前端集成步骤

#### 1. 引入验证码脚本

```html
<!-- 在页面中引入验证码客户端脚本 -->
<script src="http://your-domain.com/static/js/captcha.js"></script>
```

#### 2. 初始化验证码组件

```html
<div id="captcha-container"></div>

<script>
  // 初始化验证码
  const captcha = new HJTPXCaptcha({
    container: '#captcha-container',
    apiServer: 'http://your-domain.com/api/v1',
    captchaType: 'slider',  // slider | click | image | puzzle
    onSuccess: function(token) {
      // 验证成功，提交表单
      console.log('验证成功, Token:', token);
      // 在这里提交表单
    },
    onError: function(error) {
      console.error('验证失败:', error);
    }
  });

  // 加载验证码
  captcha.load();
</script>
```

#### 3. 滑块验证集成

```javascript
// 滑块验证码
const sliderCaptcha = new HJTPXCaptcha({
  container: '#slider-container',
  captchaType: 'slider',
  imageWidth: 320,
  imageHeight: 160,
  onSuccess: function(result) {
    // result: { token, sessionId, ... }
    submitForm(result.token);
  }
});
sliderCaptcha.load();
```

#### 4. 点选验证集成

```javascript
// 点选验证码
const clickCaptcha = new HJTPXCaptcha({
  container: '#click-container',
  captchaType: 'click',
  imageWidth: 300,
  imageHeight: 200,
  maxPoints: 3,
  onSuccess: function(result) {
    submitForm(result.token);
  }
});
clickCaptcha.load();
```

#### 5. 无感验证（静默模式）

```javascript
// 无感验证 - 用户无感知自动完成
const seamlessCaptcha = new HJTPXCaptcha({
  container: '#captcha-container',
  captchaType: 'seamless',
  onSuccess: function(result) {
    // 自动验证成功，直接通过
    console.log('静默验证通过');
  },
  onFallback: function() {
    // 降级到普通验证
    showNormalCaptcha();
  }
});
seamlessCaptcha.load();
```

### 验证结果处理

```javascript
// 验证成功后的处理
function submitForm(captchaToken) {
  // 将 token 添加到表单
  const form = document.getElementById('login-form');
  const input = document.createElement('input');
  input.type = 'hidden';
  input.name = 'captcha_token';
  input.value = captchaToken;
  form.appendChild(input);

  // 提交表单
  form.submit();
}
```

## 管理后台使用

### 登录管理后台

1. 访问 http://your-domain.com/admin
2. 使用默认账号登录：`admin` / `admin123`
3. **首次使用请立即修改密码**

### 仪表盘概览

管理后台首页显示系统关键指标：

- **今日验证量**：当天总验证次数
- **成功率**：验证通过比例
- **风险分布**：低/中/高/严重风险占比
- **实时QPS**：当前每秒请求数

### 应用管理

#### 创建应用

1. 进入「应用管理」→「创建应用」
2. 填写应用名称和描述
3. 系统自动生成 `App Key` 和 `App Secret`
4. **重要：请妥善保存 App Secret，它只会显示一次**

#### 配置应用

```json
{
  "captcha_types": ["slider", "click"],
  "risk_threshold": 50,
  "max_attempts": 5,
  "session_timeout": 300,
  "enable_behavior_analysis": true,
  "enable_ip_blacklist": true
}
```

### 验证日志查询

1. 进入「日志管理」→「验证日志」
2. 支持多条件筛选：
   - 时间范围
   - 验证码类型
   - 验证状态
   - 风险等级
   - IP地址
3. 可导出CSV/Excel格式

### 风控规则配置

1. 进入「风控规则」→「规则管理」
2. 查看系统内置规则
3. 可自定义添加新规则
4. 规则参数说明：
   - **风险分数**：触发该规则增加的分数
   - **启用状态**：是否启用该规则

### 实时监控

1. 进入「实时监控」页面
2. 查看实时验证数据
3. WebSocket自动更新
4. 支持大屏展示模式

## SDK集成示例

### Go SDK

```go
package main

import (
    "fmt"
    "github.com/opphk/hjtpx-sdk-go"
)

func main() {
    // 初始化客户端
    client := captcha.NewClient("http://localhost:8080", "your-app-key", "your-app-secret")

    // 获取滑块验证码
    resp, err := client.GetSliderCaptcha(320, 160, 8)
    if err != nil {
        panic(err)
    }
    fmt.Printf("SessionID: %s\n", resp.SessionID)

    // 验证滑块位置
    result, err := client.VerifySlider(resp.SessionID, 185, 100, trajectory)
    if err != nil {
        panic(err)
    }

    if result.Success {
        fmt.Println("验证成功")
    } else {
        fmt.Printf("验证失败: %s\n", result.Message)
    }
}
```

### Python SDK

```python
from captcha_sdk import CaptchaClient

# 初始化客户端
client = CaptchaClient(
    api_server='http://localhost:8080',
    app_key='your-app-key',
    app_secret='your-app-secret'
)

# 获取滑块验证码
resp = client.get_slider_captcha(width=320, height=160, tolerance=8)
print(f"SessionID: {resp.session_id}")

# 验证滑块
result = client.verify_slider(resp.session_id, x=185, y=100, trajectory=trajectory)
if result.success:
    print("验证成功")
else:
    print(f"验证失败: {result.message}")
```

## 常见使用场景

### 场景1：用户登录保护

```html
<!-- 登录表单 -->
<form id="login-form" action="/login" method="POST">
  <input type="text" name="username" placeholder="用户名">
  <input type="password" name="password" placeholder="密码">

  <!-- 验证码容器 -->
  <div id="captcha-container"></div>

  <button type="submit">登录</button>
</form>

<script>
// 页面加载后初始化验证码
document.addEventListener('DOMContentLoaded', function() {
  const captcha = new HJTPXCaptcha({
    container: '#captcha-container',
    onSuccess: function(token) {
      // 自动提交表单
      const form = document.getElementById('login-form');
      // 添加隐藏字段
      let input = form.querySelector('input[name="captcha_token"]');
      if (!input) {
        input = document.createElement('input');
        input.type = 'hidden';
        input.name = 'captcha_token';
        form.appendChild(input);
      }
      input.value = token;
      // 提交表单
      form.submit();
    }
  });
  captcha.load();
});
</script>
```

### 场景2：注册表单保护

```javascript
// 注册表单验证码
const captcha = new HJTPXCaptcha({
  container: '#reg-captcha',
  captchaType: 'click',  // 使用点选验证增加安全性
  onSuccess: function(result) {
    document.getElementById('reg-form').submit();
  }
});
captcha.load();
```

### 场景3：评论/发帖保护

```javascript
// 评论保护 - 使用拼图验证
const captcha = new HJTPXCaptcha({
```

### 场景4：无感验证（已信任用户）

```javascript
const seamlessCaptcha = new HJTPXCaptcha({
```

## 最佳实践

### 安全性最佳实践

1. **使用HTTPS**
   - 生产环境必须使用HTTPS
   - 防止Token被中间人拦截

2. **验证结果服务端校验**
   ```go
   // 后端必须验证Token有效性
   func LoginHandler(c *gin.Context) {
       token := c.PostForm("captcha_token")
       if token == "" {
           c.JSON(400, gin.H{"error": "验证码必填"})
           return
       }

       // 验证Token
       valid, err := captchaClient.VerifyToken(token)
       if err != nil || !valid {
           c.JSON(400, gin.H{"error": "验证码验证失败"})
           return
       }

       // 继续处理登录逻辑...
   }
   ```

3. **限制验证频率**
   - 同一IP每分钟最多验证5次
   - 失败3次后需要等待5分钟

4. **日志记录**
   - 记录所有验证尝试
   - 监控异常模式

### 性能最佳实践

1. **验证码缓存**
   - 已生成的验证码在Redis中缓存5分钟
   - 避免重复生成

2. **图片预加载**
   ```javascript
   // 用户鼠标悬停在登录框时预加载验证码
   document.getElementById('login-form').addEventListener('mouseenter', function() {
     if (!captchaLoaded) {
       captcha.load();
       captchaLoaded = true;
     }
   });
   ```

3. **异步验证**
   ```javascript
   // 用户滑动完成后，先展示loading，同时异步验证
   slider.on('slide', function(data) {
     showLoading();
     verifyAsync(data).then(result => {
       hideLoading();
       handleResult(result);
     });
   });
   ```

### 用户体验最佳实践

1. **渐进式验证**
   - 新用户/高风险操作：使用复杂验证
   - 老用户/低风险操作：使用简单验证或无感验证

2. **清晰的错误提示**
   ```javascript
   captcha.onError = function(error) {
     showMessage({
       type: 'error',
       message: '验证失败，请重试',
       duration: 3000
     });
   };
   ```

3. **验证成功动画**
   ```javascript
   captcha.onSuccess = function() {
     showSuccessAnimation().then(() => {
       proceedToNextStep();
     });
   };
   ```

4. **移动端适配**
   - 验证码图片响应式缩放
   - 触摸滑动支持
   - 更大的点击区域

### 配置推荐

| 场景 | 验证类型 | 风险阈值 | 最大尝试 |
|------|----------|----------|----------|
| 登录 | slider | 50 | 3 |
| 注册 | click | 40 | 3 |
| 评论 | image | 60 | 5 |
| 找回密码 | slider+click | 30 | 2 |
| 高价值操作 | puzzle | 40 | 2 |
