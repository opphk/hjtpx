## 系统架构

### 整体架构

```
┌─────────────────────────────────────────────────┐
│                    Nginx                         │
│              (反向代理 + 负载均衡)                │
└─────────────────────────────────────────────────┘
                        │
┌─────────────────────────────────────────────────┐
│                 Backend (Go)                     │
│  ┌──────────────┬──────────────┬──────────────┐  │
│  │   Handler    │  Middleware  │   Service   │  │
│  └──────────────┴──────────────┴──────────────┘  │
│  ┌──────────────┬──────────────┬──────────────┐  │
│  │   Router     │    Cache     │   Models    │  │
│  └──────────────┴──────────────┴──────────────┘  │
└─────────────────────────────────────────────────┘
          │                    │
┌──────────────────┐  ┌──────────────────┐
│    PostgreSQL     │  │      Redis        │
│    (主数据库)      │  │    (缓存/会话)    │
└──────────────────┘  └──────────────────┘
```

### 架构说明

| 层级 | 说明 |
|------|------|
| Nginx | 反向代理、SSL终结、负载均衡、静态资源服务 |
| Handler | HTTP请求处理、参数验证、响应格式化 |
| Middleware | 认证授权、限流、日志、安全防护 |
| Service | 业务逻辑处理、数据处理、缓存逻辑 |
| Repository | 数据访问层、数据库操作 |
| Cache | Redis缓存、会话管理 |
| Database | PostgreSQL主数据库 |

## 核心模块

### 1. Handler 层

负责处理 HTTP 请求，是系统的入口层。

**主要职责：**
- 接收和解析 HTTP 请求
- 参数验证和转换
- 调用 Service 层处理业务
- 格式化响应数据

**实现位置：**
- `backend/internal/api/handler/`

**核心 Handler：**

| Handler | 说明 |
|---------|------|
| auth.go | 用户认证（登录、注册、Token刷新） |
| captcha.go | 验证码生成和验证入口 |
| slider_captcha.go | 滑块验证码处理 |
| click_captcha.go | 点选验证码处理 |
| image_captcha.go | 图形验证码处理 |
| application.go | 应用管理 |
| blacklist.go | 黑名单管理 |
| logs.go | 日志管理 |
| stats.go | 统计数据 |

### 2. Middleware 层

负责横切关注点的处理。

**主要职责：**
- 认证授权
- 请求限流
- 日志记录
- 安全防护（CORS、CSRF、XSS、SQL注入）
- 错误处理
- 性能监控

**实现位置：**
- `backend/internal/api/middleware/`

**核心 Middleware：**

| Middleware | 说明 |
|------------|------|
| auth.go | JWT Token 验证 |
| rate_limit.go | 请求限流 |
| logger.go | 请求日志记录 |
| cors.go | 跨域资源共享 |
| csrf.go | CSRF 防护 |
| xss.go | XSS 防护 |
| sql_injection.go | SQL注入防护 |
| ip_filter.go | IP过滤 |
| whitelist.go | IP白名单 |
| blacklist.go | IP黑名单 |
| recovery.go | 异常恢复 |
| security_log.go | 安全日志 |

### 3. Service 层

负责业务逻辑处理。

**主要职责：**
- 业务逻辑实现
- 数据处理和转换
- 缓存逻辑处理
- 第三方服务调用

**实现位置：**
- `backend/internal/service/`

**核心 Service：**

| Service | 说明 |
|---------|------|
| user_service.go | 用户管理 |
| application_service.go | 应用管理 |
| blacklist_service.go | 黑名单管理 |
| behavior_analysis.go | 行为分析 |
| cache_service.go | 缓存服务 |
| rate_limit_service.go | 限流服务 |
| detection_service.go | 风险检测 |
| stats_service.go | 统计服务 |
| log_service.go | 日志服务 |

### 4. Repository 层

负责数据访问。

**主要职责：**
- 数据库操作
- 数据模型映射
- 事务管理

**实现位置：**
- `backend/pkg/models/`
- `backend/pkg/database/`

### 5. 缓存层

基于 Redis 实现多级缓存。

**缓存策略：**
- Session 缓存
- 验证码缓存
- 限流计数器
- 热点数据缓存

## 数据流设计

### 验证码生成流程

```
用户请求
    ↓
Handler: GenerateSliderCaptcha
    ↓
Service: 生成验证码数据
    ↓
Cache: 存储验证码Session
    ↓
返回验证码图片和SessionID
```

### 验证码验证流程

```
用户提交验证
    ↓
Handler: VerifyCaptcha
    ↓
Middleware: Auth、RateLimit、Blacklist
    ↓
Service: 验证逻辑
    ├── 1. 检查Session有效性
    ├── 2. 验证坐标准确性
    ├── 3. 行为分析
    └── 4. 风险评分
    ↓
Cache: 更新验证结果
    ↓
记录日志
    ↓
返回验证结果
```

### 请求处理流程

```
HTTP请求
    ↓
Nginx: 反向代理/SSL
    ↓
Middleware: CORS、CSRF、RateLimit
    ↓
Handler: 路由匹配
    ↓
Middleware: Auth、Validation
    ↓
Service: 业务逻辑
    ↓
Repository: 数据访问
    ↓
Cache/DB: 数据存储
    ↓
返回响应
```

## 性能优化

### 1. 缓存策略

**多级缓存：**
- L1: 进程内缓存（sync.Pool）
- L2: Redis 分布式缓存

**缓存预热：**
- 启动时预加载热点数据
- 定时刷新热点数据

**缓存失效策略：**
- TTL 自动过期
- 主动删除
- LRU 淘汰

### 2. 数据库优化

**索引优化：**
- 为常用查询字段添加索引
- 复合索引优化
- 定期分析慢查询

**查询优化：**
- 避免 N+1 查询
- 使用预编译语句
- 分页查询优化

**连接池优化：**
- 合理配置连接池大小
- 连接复用
- 超时控制

### 3. 并发优化

**Worker Pool：**
- 限制并发数
- 任务队列管理
- 优雅关闭

**Channel Buffering：**
- 减少阻塞
- 批量处理

**sync.Pool：**
- 对象复用
- 减少GC压力

### 4. HTTP 优化

**Nginx 优化：**
- keepalive 连接复用
- gzip 压缩
- 静态资源缓存
- 负载均衡策略

**客户端优化：**
- HTTP/2 支持
- 连接复用
- 请求合并
