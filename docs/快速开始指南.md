## 环境准备

### 方式一：使用Docker（推荐）

仅需安装Docker和Docker Compose即可，适合快速体验和开发环境。

```bash
# 检查Docker版本
docker --version
docker-compose --version

# 如未安装，请参考官方文档安装
# https://docs.docker.com/get-docker/
# https://docs.docker.com/compose/install/
```

### 方式二：手动安装

需要准备以下环境：

| 组件 | 最低版本 | 安装命令参考 |
|------|---------|-------------|
| Go | 1.21+ | `wget https://go.dev/dl/go1.21.linux-amd64.tar.gz` |
| PostgreSQL | 12+ | `apt-get install postgresql` |
| Redis | 6+ | `apt-get install redis-server` |
| Git | 任意 | `apt-get install git` |

## 快速安装

### 步骤1：获取代码

```bash
git clone https://github.com/opphk/hjtpx.git
cd hjtpx
```

### 步骤2：配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑配置文件（必填项：数据库密码、JWT密钥）
vim .env
```

必填配置项说明：

```bash
# 数据库配置
POSTGRES_PASSWORD=your-secure-password    # 必须修改，使用强密码
POSTGRES_DB=hjtpx_db                      # 数据库名，可自定义

# Redis配置
REDIS_PASSWORD=your-redis-password       # 如需密码则设置

# JWT配置
JWT_SECRET=your-very-secure-jwt-secret   # 必须修改，32位以上
```

### 步骤3：一键启动

```bash
# 启动所有服务（PostgreSQL、Redis、应用服务、监控组件）
docker-compose up -d

# 等待服务启动（约10-30秒）
sleep 30

# 查看服务状态
docker-compose ps
```

正常情况下，应看到所有服务状态为 `Up`：

```
NAME                STATUS
hjtpx-postgres-1    Up
hjtpx-redis-1       Up
hjtpx-app-1         Up
hjtpx-prometheus-1  Up
hjtpx-grafana-1     Up
```

## 基本配置

### Docker Compose 方式

修改 `.env` 文件即可，主要配置项：

```bash
# 服务端口
SERVER_PORT=8080

# PostgreSQL
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_DB=hjtpx_db
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your-password

# Redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=

# JWT
JWT_SECRET=your-secret-key-change-me
JWT_EXPIRE_HOURS=24
```

### 手动部署方式

配置文件位置：`backend/config/config.yaml`

```yaml
server:
  port: 8080
  mode: release

postgres:
  host: localhost
  port: 5432
  user: postgres
  password: "your-password"
  db_name: hjtpx_db
  sslmode: disable

redis:
  host: localhost
  port: 6379
  password: ""
  db: 0

jwt:
  secret: "your-secret-key-change-me"
  expire_hours: 24
```

## 服务启动

### Docker Compose 启动

```bash
# 启动所有服务
docker-compose up -d

# 启动并重新构建
docker-compose up -d --build

# 查看日志
docker-compose logs -f app

# 停止服务
docker-compose down

# 停止并删除数据卷（慎用，会清除数据）
docker-compose down -v
```

### 手动启动

```bash
# 进入后端目录
cd backend

# 下载依赖
go mod download

# 编译
go build -o hjtpx ./cmd/api/main.go

# 启动服务
./hjtpx

# 或使用环境变量启动
POSTGRES_HOST=localhost REDIS_HOST=localhost ./hjtpx
```

## 首次使用

### 登录管理后台

1. 访问 http://localhost/admin
2. 输入默认账号：
   - 用户名：`admin`
   - 密码：`admin123`
3. 首次登录后**立即修改密码**

### 创建应用

1. 登录后进入「应用管理」
2. 点击「创建应用」
3. 填写应用信息：
   - 应用名称：填写你的应用名称
   - 应用域名：填写使用验证码的域名
   - API密钥：系统自动生成

### 获取API密钥

创建应用后，在应用详情页复制API密钥：

```
API Key: your-api-key-here
API Secret: your-api-secret-here
```

## 集成示例

### 前端集成示例

在网页中嵌入验证码：

```html
<!DOCTYPE html>
<html>
<head>
    <title>验证码集成示例</title>
    <style>
        #captcha-container {
            width: 320px;
            margin: 50px auto;
        }
    </style>
</head>
<body>
    <div id="captcha-container"></div>
    
    <script src="http://localhost:8080/static/js/captcha.js"></script>
    <script>
        // 初始化验证码
        HJTPXCaptcha.init({
            container: '#captcha-container',
            apiServer: 'http://localhost:8080',
            apiKey: 'your-api-key',
            onSuccess: function(token) {
                console.log('验证成功，Token:', token);
                // 将token发送到你的服务器进行验证
            },
            onError: function(error) {
                console.error('验证失败:', error);
            }
        });
    </script>
</body>
</html>
```

### 后端验证示例

用户提交验证后，后端验证token有效性：

```go
package main

import (
    "fmt"
    "net/http"
    "encoding/json"
)

type VerifyRequest struct {
    Token    string `json:"token"`
    SessionID string `json:"session_id"`
}

func main() {
    // 接收前端提交的验证请求
    http.HandleFunc("/api/verify", func(w http.ResponseWriter, r *http.Request) {
        var req VerifyRequest
        json.NewDecoder(r.Body).Decode(&req)
        
        // 调用HJTPX API验证
        verifyURL := "http://localhost:8080/api/v1/captcha/slider/verify"
        resp, _ := http.Post(verifyURL, "application/json", 
            strings.NewReader(fmt.Sprintf(`{"session_id":"%s","token":"%s"}`, 
                req.SessionID, req.Token)))
        
        // 处理响应...
    })
    
    http.ListenAndServe(":8081", nil)
}
```

### 使用Go SDK

```go
package main

import (
    "fmt"
    "github.com/opphk/hjtpx/sdk/go"
)

func main() {
    client := captcha.NewClient(
        "http://localhost:8080",
        "your-api-key",
        "your-api-secret",
    )
    
    // 生成滑块验证码
    result, err := client.GenerateSliderCaptcha(320, 160, 5)
    if err != nil {
        panic(err)
    }
    
    fmt.Printf("SessionID: %s\n", result.SessionID)
    fmt.Printf("背景图: %s\n", result.BackgroundImage[:50])
    
    // 验证滑块位置
    verifyResult, err := client.VerifySliderCaptcha(
        result.SessionID,
        150,  // 滑块X坐标
        0.95, // 相似度阈值
    )
    if err != nil {
        panic(err)
    }
    
    fmt.Printf("验证结果: %v\n", verifyResult.Success)
}
```

## 常见问题

### Q: 服务启动失败怎么办？

```bash
# 查看详细日志
docker-compose logs -f

# 常见原因：
# 1. 端口被占用 - 检查8080端口是否被占用
lsof -i :8080

# 2. 数据库连接失败 - 检查PostgreSQL是否正常运行
docker-compose ps postgres

# 3. Redis连接失败 - 检查Redis是否正常运行
docker-compose ps redis
```

### Q: 如何修改管理员密码？

1. 登录管理后台 http://localhost/admin
2. 进入「用户管理」
3. 点击「修改密码」

或通过API：

```bash
curl -X POST http://localhost:8080/api/v1/user/change-password \
  -H "Authorization: Bearer your-jwt-token" \
  -H "Content-Type: application/json" \
  -d '{"old_password": "admin123", "new_password": "your-new-password"}'
```

### Q: 如何查看API文档？

- Swagger UI: http://localhost:8080/swagger/
- 在线API文档: `docs/api-docs.html`
- Postman集合: `docs/postman-collection.json`
