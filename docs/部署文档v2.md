## 系统要求

### 硬件要求

| 环境 | CPU | 内存 | 磁盘 |
|------|-----|------|------|
| 开发环境 | 2核 | 4GB | 20GB |
| 预发布环境 | 4核 | 8GB | 50GB |
| 生产环境 | 8核+ | 16GB+ | 100GB+ |

### 软件要求

- Go 1.21+
- Node.js 18+
- PostgreSQL 14+
- Redis 7+
- Docker 24+
- Kubernetes 1.28+ (可选)

## 部署方式

### 方式一：Docker Compose（推荐用于开发环境）

```bash
# 克隆代码
git clone https://github.com/hjtpx/hjtpx.git
cd hjtpx

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f api
```

### 方式二：Kubernetes（推荐用于生产环境）

```bash
# 应用配置
kubectl apply -f k8s/

# 检查部署状态
kubectl get pods -n hjtpx

# 查看日志
kubectl logs -f deployment/hjtpx -n hjtpx
```

## 开发环境部署

### 1. 安装依赖

```bash
# 安装Go
wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.0.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

# 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装PostgreSQL
sudo apt-get install -y postgresql postgresql-contrib

# 安装Redis
sudo apt-get install -y redis-server
```

### 2. 配置数据库

```bash
# 创建数据库
sudo -u postgres psql << EOF
CREATE DATABASE hjtpx;
CREATE USER hjtpx_user WITH ENCRYPTED PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE hjtpx TO hjtpx_user;
\c hjtpx
GRANT ALL ON SCHEMA public TO hjtpx_user;
EOF

# 初始化数据库
cd backend
go run cmd/api/main.go migrate
```

### 3. 配置Redis

```bash
# 启动Redis
redis-server --daemonize yes

# 测试连接
redis-cli ping
```

### 4. 编译运行

```bash
# 编译后端
cd backend
go build -o hjtpx ./cmd/api

# 启动服务
./hjtpx --config config.yaml
```

### 5. 构建前端

```bash
# 安装依赖
cd frontend
npm install

# 开发模式
npm run dev

# 生产构建
npm run build
```

## Docker部署

### 构建镜像

```bash
# 构建镜像
docker build -t hjtpx:latest .

# 查看镜像
docker images hjtpx
```

### 运行容器

```bash
# 运行容器
docker run -d \
  --name hjtpx \
  -p 8080:8080 \
  -e DATABASE_URL=postgres://user:pass@host:5432/hjtpx \
  -e REDIS_URL=redis://host:6379 \
  -e API_KEY=your-api-key \
  hjtpx:latest

# 查看日志
docker logs -f hjtpx

# 停止容器
docker stop hjtpx
```

### Docker Compose配置

```yaml
version: '3.8'

services:
  api:
    build: .
    ports:
      - "8080:8080"
    environment:
      - DATABASE_URL=postgres://hjtpx_user:password@db:5432/hjtpx
      - REDIS_URL=redis://redis:6379
      - API_KEY=${API_KEY}
    depends_on:
      - db
      - redis
    restart: unless-stopped

  db:
    image: postgres:14
    environment:
      - POSTGRES_DB=hjtpx
      - POSTGRES_USER=hjtpx_user
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
```

## Kubernetes部署

### 创建命名空间

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: hjtpx
  labels:
    name: hjtpx
```

### 配置映射

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: hjtpx-config
  namespace: hjtpx
data:
  config.yaml: |
    server:
      host: 0.0.0.0
      port: 8080
    database:
      url: postgres://hjtpx_user:password@postgres:5432/hjtpx
    redis:
      url: redis://redis:6379
    captcha:
      max_attempts: 5
      session_ttl: 300
```

### 部署服务

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hjtpx
  namespace: hjtpx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: hjtpx
  template:
    metadata:
      labels:
        app: hjtpx
    spec:
      containers:
        - name: hjtpx
          image: ghcr.io/hjtpx/hjtpx:latest
          ports:
            - containerPort: 8080
          envFrom:
            - configMapRef:
                name: hjtpx-config
            - secretRef:
                name: hjtpx-secrets
          resources:
            requests:
              memory: "256Mi"
              cpu: "250m"
            limits:
              memory: "1Gi"
              cpu: "1000m"
          livenessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 30
            periodSeconds: 10
          readinessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 5
            periodSeconds: 5
```

### 金丝雀部署

```bash
# 创建金丝雀部署
kubectl create deployment hjtpx-canary --image=ghcr.io/hjtpx/hjtpx:canary -n hjtpx

# 流量分割
kubectl apply -f - <<EOF
apiVersion: flagger.app/v1beta1
kind: Canary
metadata:
  name: hjtpx
  namespace: hjtpx
spec:
  targetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: hjtpx
  canaryDeployment:
    replicas: 1
    image: ghcr.io/hjtpx/hjtpx:canary
  analysis:
    interval: 1m
    threshold: 5
    maxWeight: 50
    stepWeight: 10
EOF
```

### 回滚机制

```bash
# 查看部署历史
kubectl rollout history deployment/hjtpx -n hjtpx

# 回滚到上一个版本
kubectl rollout undo deployment/hjtpx -n hjtpx

# 回滚到指定版本
kubectl rollout undo deployment/hjtpx -n hjtpx --to-revision=2
```

## 备份与恢复

### 数据库备份

```bash
# 备份
pg_dump -h localhost -U hjtpx_user hjtpx > backup_$(date +%Y%m%d).sql

# 压缩备份
pg_dump -h localhost -U hjtpx_user hjtpx | gzip > backup_$(date +%Y%m%d).sql.gz

# 自动备份脚本
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
pg_dump -h localhost -U hjtpx_user hjtpx | gzip > /backups/hjtpx_$DATE.sql.gz
find /backups -name "hjtpx_*.sql.gz" -mtime +7 -delete
```

### 数据恢复

```bash
# 恢复数据
gunzip < backup_20240101.sql.gz | psql -h localhost -U hjtpx_user hjtpx

# 指定时间点恢复
pg_restore -h localhost -U hjtpx_user -d hjtpx --point-in-time="2024-01-01 12:00:00"
```

## 故障排查

### 常见问题

#### 1. 服务启动失败

```bash
# 检查配置
./hjtpx --validate-config

# 检查日志
journalctl -u hjtpx -n 100

# 检查依赖服务
curl http://localhost:5432
curl http://localhost:6379
```

#### 2. 数据库连接失败

```bash
# 测试连接
psql -h localhost -U hjtpx_user -d hjtpx

# 检查连接数
SELECT count(*) FROM pg_stat_activity;
```

#### 3. 内存使用过高

```bash
# 查看内存使用
ps aux | grep hjtpx

# 设置内存限制
./hjtpx --max-memory=512MB
```

### 性能优化

```bash
# 启用缓存
redis-cli config set maxmemory 1gb

# 优化数据库
psql -h localhost -U hjtpx_user -d hjtpx -c "VACUUM ANALYZE;"

# 重启服务
systemctl restart hjtpx
```

## 安全配置

### HTTPS配置

```nginx
server {
    listen 443 ssl http2;
    server_name example.com;

    ssl_certificate /etc/ssl/certs/example.com.crt;
    ssl_certificate_key /etc/ssl/private/example.com.key;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 防火墙配置

```bash
# 开放必要端口
ufw allow 22/tcp    # SSH
ufw allow 80/tcp     # HTTP
ufw allow 443/tcp    # HTTPS
ufw allow 8080/tcp  # API

# 启用防火墙
ufw enable
```
