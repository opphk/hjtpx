# 行为验证系统 - 开发核心文档 v3.0

## 项目概述

开发一个完整的行为验证系统，前后端均使用Go语言，目标是超越极验、易盾、五秒盾等现有产品。

### 项目定位

- **核心理念**：提供高性能、高安全性的行为验证服务
- **目标用户**：网站/应用开发者，需要防止机器人攻击的场景
- **竞争优势**：更低的响应延迟、更高的安全性、更易用的SDK

## 核心特性

- **多种验证方式**：滑块验证、点选验证、图形验证码
- **智能行为分析**：AI驱动的鼠标轨迹分析、风险评分系统
- **用户端**：简洁易用的验证界面
- **管理端**：完整的验证数据统计、配置管理
- **高安全性**：多层防护体系，有效抵御机器攻击
- **高可扩展性**：模块化设计，支持多种部署方式

## 开发规范

### CSS 使用规范（硬性要求）

- **所有 CSS 必须从 `https://www.bootcdn.cn` 加载，禁止编写任何自定义 CSS 代码**
- 允许使用 `<style>` 内联标签编写少量布局辅助样式（如 flex、margin、padding 等 Bootstrap 无法直接覆盖的场景），但不得编写完整的 UI 样式
- 项目中不得存在任何 `.css` 文件
- 优先使用 Bootstrap 5 内置的 utility classes 实现布局和样式

## 常见问题

### Q: 如何修改管理员密码？
A: 登录管理后台，在"系统设置"中修改密码。

### Q: 如何添加自定义验证码背景？
A: 在`backend/internal/api/handler/captcha.go`中修改图片生成逻辑。

### Q: 如何接入现有用户系统？
A: 使用SDK集成，参考`sdk/go/examples/demo.go`示例。

### Q: 如何查看验证码访问统计？
A: 登录管理后台，访问"统计"页面查看详细数据。

### Q: 如何配置API限流规则？
A: 在`backend/config/config.yaml`中配置`rate_limit`部分。

### Q: 如何启用HTTPS？
A: 在`nginx/nginx.conf`中配置SSL证书。
