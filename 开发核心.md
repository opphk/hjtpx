## 高可用架构 (v4.0-HA) ✅

### 架构概览

系统采用多节点部署架构，支持水平扩展和自动故障转移，确保服务的高可用性和数据一致性。

```
┌─────────────────────────────────────────────────────────────┐
│                      Load Balancer (Nginx)                  │
│                     ┌─────────┬─────────┐                   │
│                     │ Round   │ Least  │                   │
│                     │ Robin   │ Conn   │                   │
│                     └─────────┴─────────┘                   │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
   ┌─────────┐          ┌─────────┐          ┌─────────┐
   │ Node-1  │          │ Node-2  │          │ Node-3  │
   │(Primary)│◄────────►│(Secondary)│◄───────►│(Secondary)│
   └─────────┘   Gossip  └─────────┘   Gossip  └─────────┘
        │                                      │
        └──────────────────────────────────────┘
                      Cluster State
```

### 核心组件

#### 1. 健康检查模块 (Health Checker)

- **主动健康检查**: 定期向所有节点发送HTTP健康检查请求
- **多维度检测**: 支持HTTP/TCP/自定义检查函数
- **滚动窗口统计**: 基于滑动窗口的成功率计算
- **健康评分**: 综合延迟和成功率计算节点健康分数
- **状态变更通知**: 节点状态变化时触发回调

**API端点**:
- `GET /health/ha` - 高可用健康检查
- `GET /health/live` - 存活检查
- `GET /health/ready` - 就绪检查
- `GET /health/cluster` - 集群状态

#### 2. 故障转移控制器 (Failover Controller)

- **自动故障转移**: 检测到主节点故障时自动切换
- **多策略支持**: 自动/手动/定时故障转移
- **熔断机制**: 防止故障扩散
- **自动恢复**: 故障节点恢复后自动重新加入
- **事件日志**: 完整的故障转移事件记录

**配置参数**:
- `failure_threshold`: 故障阈值（默认3次）
- `recovery_threshold`: 恢复阈值（默认2次）
- `failover_timeout`: 故障转移超时（默认30秒）
- `enable_auto_recovery`: 启用自动恢复

#### 3. 数据同步服务 (Data Sync Service)

- **多种同步策略**:
  - Eager（同步）: 写入时立即同步到所有节点
  - Lazy（异步）: 后台异步同步
  - Eventual（最终一致）: 保证最终一致性
  - Quorum（仲裁）: 多数节点确认即可

- **冲突解决**: 版本向量时钟管理冲突
- **增量同步**: 支持全量/增量/差异同步
- **压缩传输**: 可选的Gzip压缩
- **重试机制**: 自动重试失败同步

#### 4. 集群管理器 (Cluster Manager)

- **Raft共识**: 基于Raft算法的leader选举
- **Gossip协议**: 节点间状态传播
- **成员管理**: 动态加入/离开集群
- **分区检测**: 检测并处理网络分区

**角色**:
- Leader: 处理所有写请求
- Follower: 接收leader复制
- Candidate: 选举时的候选节点

#### 5. 高可用负载均衡器 (HA Proxy)

- **多种负载均衡策略**:
  - Round Robin（轮询）
  - Weighted Round Robin（加权轮询）
  - Least Connections（最少连接）
  - IP Hash（IP哈希）
  - Consistent Hash（一致性哈希）

- **熔断器**: 防止故障级联
- **重试机制**: 请求失败自动重试
- **健康检查集成**: 自动跳过不健康节点

### 多节点部署配置

#### Docker Compose 集群模式

```bash
# 启动集群模式（3节点）
CLUSTER_MODE=true NODE_COUNT=3 ./scripts/cluster-deploy.sh

# 启动单节点模式
docker-compose up -d
```

#### 节点配置

```yaml
app-node1:
  environment:
    - NODE_ID=node-1
    - NODE_ROLE=primary
    - CLUSTER_ENABLED=true
    - HA_ENABLED=true
    - FAILOVER_ENABLED=true

app-node2:
  environment:
    - NODE_ID=node-2
    - NODE_ROLE=secondary
    - CLUSTER_SEEDS=node1:7946
```

### 负载均衡配置 (Nginx)

```
upstream backend_http {
    least_conn;

    server app-node1:8080 weight=100 max_fails=3 fail_timeout=30s;
    server app-node2:8080 weight=90 max_fails=3 fail_timeout=30s;
    server app-node3:8080 weight=90 max_fails=3 fail_timeout=30s;

    keepalive 64;
}
```
